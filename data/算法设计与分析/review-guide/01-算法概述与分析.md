# 第 1 章 算法概述与分析

## 1. 本章概述

本章介绍算法的基本概念与特性，以及算法分析的两大核心方法——**时间复杂度分析**（渐近分析、步计数法、递归方程求解）与**空间复杂度分析**。同时引入**P、NP、NPC、NPH**等复杂性类及其关系，为后续各章的算法学习奠定分析基础。

---

## 2. 知识点讲解

### 知识点 1：算法的定义与基本特性

**通俗定义**：算法是解决某一特定问题的一系列**有穷**、**确定**、**可执行**的操作步骤。可以理解为一份"烹饪食谱"——每一步都明确、可操作，且最终能做完。

**标准定义**：
- **有穷性（Finiteness）**：算法必须在执行**有限步**后终止
- **确定性（Determinism）**：算法的每一步都必须有**唯一确定**的含义，不能有歧义
- **可执行性（Effectiveness）**：算法的每一步都必须是**可行的**（原则上可以用纸笔完成）

**举例**：
- 计算 1 到 100 的和：用循环累加，每一步都是确定的加法运算，循环 100 次后终止 → 是一个算法
- "如果这个数很大就做什么"（没有定义"大"的具体标准）→ 不满足确定性

**初学易错点**：
- 不要混淆"算法"和"程序"：程序不一定满足有穷性（如操作系统永久运行），但算法**必须**有穷
- "有穷性"是指**步数**有穷，而不是时间上的"有限"（步数有穷但每步耗时不同，仍算有穷）

---

### 知识点 2：渐近记号（Asymptotic Notations）

**通俗定义**：当问题规模 $n$ 很大时，我们只关心算法运行时间的**增长趋势**，忽略常数因子和低阶项。好比只看"增长率"不看"小波动"。

**标准定义**：
- **大 $O$ 记号（上界）**：$O(g(n)) = \{ f(n) \mid \exists c>0, n_0 > 0, \text{使得 } 0 \leq f(n) \leq c g(n), \forall n \geq n_0 \}$
- **大 $\Omega$ 记号（下界）**：$\Omega(g(n)) = \{ f(n) \mid \exists c>0, n_0 > 0, \text{使得 } 0 \leq c g(n) \leq f(n), \forall n \geq n_0 \}$
- **大 $\Theta$ 记号（紧界）**：$\Theta(g(n)) = \{ f(n) \mid \exists c_1, c_2 > 0, n_0 > 0, \text{使得 } 0 \leq c_1 g(n) \leq f(n) \leq c_2 g(n), \forall n \geq n_0 \}$

**举例**：
- $f(n) = 3n^2 + 10n$，则 $f(n) = O(n^2)$（$n$ 充分大时，$3n^2+10n \leq 4n^2$）
- $f(n) = 3n^2 + 10n$，也是 $\Omega(n^2)$（$n$ 充分大时，$3n^2+10n \geq 3n^2$）
- 因此 $f(n) = \Theta(n^2)$

**初学易错点**：
- 大 $O$ 是**上界**，不是"精确值"：$n = O(n^2)$ 是**正确**的（因为 $n \leq 1\cdot n^2$ 对 $n \geq 1$ 成立），但不够"紧"
- 习惯上算法分析追求**最紧**的上界，即 $\Theta$ 或最紧的 $O$
- 渐近记号表示的是**函数的集合**，所以 $O(f) + O(g) = O(f+g)$ 是集合的加法（任意取两个函数之和仍属于 $O(f+g)$）
- 看到$O(n^3)$并不意味着执行时间就等于 $n^3$，而是与 $n^3$ **同阶或更低**

---

### 知识点 3：渐近表达式（Asymptotic Expression）

**通俗定义**：给定一个函数 $T(n)$，找一个"形态最简"的函数 $t(n)$，使得当 $n$ 趋于无穷时，$T(n)$ 与 $t(n)$ 的相对差距趋近于 0。

**标准定义**：
$$
\lim_{n \to \infty} \frac{T(n) - t(n)}{T(n)} = 0
$$
即 $t(n)$ 是 $T(n)$ 的**主项**（去掉低阶项和系数后的留下部分）。

**举例**：
- $T(n) = 3n^2 + 10n$，渐近表达式为 $3n^2$
- $T(n) = 21 + 1/n$，渐近表达式为 $21$（因为 $1/n \to 0$）

**初学易错点**：
- 渐近表达式**保留系数**（如 $3n^2$），而渐近记号**去掉系数**（如 $O(n^2)$），两者不同
- 别把"渐近表达式"和"渐近记号"混为一谈

---

### 知识点 4：时间复杂度分析——步计数法

**通俗定义**：把算法中每一条语句（基本操作）的执行次数统计出来，每条语句给一个"单步代价"（s/e，即 steps per execution），然后求和，得到一个关于 $n$ 的函数，再化简为渐近形式。

**标准分析方法**：

| 语句 | s/e | 执行频率 | 总步数 |
|------|-----|----------|--------|
| 语句1 | 1 | $f_1(n)$ | $f_1(n)$ |
| 语句2 | 1 | $f_2(n)$ | $f_2(n)$ |
| ... | ... | ... | ... |

总步数 = $\sum$ (s/e × 频率)，然后取渐近表达式。

**举例**：一个简单的 for 循环
```cpp
int sum = 0;                // s/e=1, 频率=1 → 1步
for(int i = 0; i < n; i++)  // s/e=1, 频率=n+1 → n+1步（最后一次判断失败）
    sum += i;               // s/e=1, 频率=n → n步
```
总步数 = $1 + (n+1) + n = 2n + 2$，渐近时间复杂度为 $\Theta(n)$。

**初学易错点**：
- for 循环的判断语句（`i < n`）执行了 $n+1$ 次（第 $n+1$ 次判断为 false 退出），不是 $n$ 次
- if 语句的"频率"取决于条件是否满足——最好/最坏/平均情形不同
- 步计数法的 s/e 通常取 1（因为每条语句的执行时间可视为常数），但也可根据不同操作设置不同值

---

### 知识点 5：递归方程的求解方法

**通俗定义**：很多算法（如归并排序）的执行时间可以用包含自身的形式表达，称为递归方程。解递归方程就是找出 $T(n)$ 的显式形式（或渐近表示）。

**三种主要方法**（对应 Q0069）：

#### 方法一：代入法（代入归纳法）

**步骤**：
1. 猜测 $T(n)$ 的形式（如 $T(n) \leq c_1 n^2 - c_2 n$）
2. 用数学归纳法证明猜测成立
3. 确定常数条件

**举例**：
$T(1) = c$，$T(n) = 4T(n/2) + n \quad (n > 1)$
猜测 $T(n) \leq c_1 n^2 - c_2 n$，代入：
$$
T(n) = 4T(n/2) + n \leq 4[c_1 (n/2)^2 - c_2 (n/2)] + n = c_1 n^2 - 2c_2 n + n = c_1 n^2 - c_2 n - (c_2 n - n)
$$
当 $c_2 > 1$ 时，$c_2 n - n > 0$，所以 $T(n) \leq c_1 n^2 - c_2 n$。再取 $c_1 > c + c_2$ 使 $n=1$ 时也成立。

#### 方法二：递归树法

**步骤**：
1. 将递归式画成树形结构
2. 计算每层的工作量之和
3. 对所有层求和

**举例**（Q0034）：$T(n) = T(n-1) + cn$

展开为递归树：
- 第 0 层（根）：$cn$
- 第 1 层：$c(n-1)$
- 第 2 层：$c(n-2)$
- ...
- 第 $n-1$ 层：$c \cdot 1$
- 第 $n$ 层：$T(0)$（常数）

求和：
$$
T(n) = \sum_{k=1}^{n} ck + T(0) = c \cdot \frac{n(n+1)}{2} + T(0) = \Theta(n^2)
$$

#### 方法三：主定理（Master Theorem）

**标准形式**：$T(n) = a T(n/b) + f(n)$，其中 $a \geq 1$，$b > 1$

**三种情形**（比较 $f(n)$ 与 $n^{\log_b a}$）：

1. **情形 1**：$f(n) = O(n^{\log_b a - \varepsilon})$，$\varepsilon > 0$ → $T(n) = \Theta(n^{\log_b a})$
2. **情形 2**：$f(n) = \Theta(n^{\log_b a})$ → $T(n) = \Theta(n^{\log_b a} \log n)$
3. **情形 3**：$f(n) = \Omega(n^{\log_b a + \varepsilon})$，$\varepsilon > 0$，且 $a f(n/b) \leq c f(n)$（$c < 1$） → $T(n) = \Theta(f(n))$

**举例**：
- $T(n) = 7T(n/2) + cn^2$：$a=7, b=2, \log_b a \approx 2.81, f(n)=cn^2 = O(n^{2.81-\varepsilon})$ → 情形 1，$T(n) = \Theta(n^{2.81})$
- $T(n) = 4T(n/2) + n^2$：$a=4, b=2, \log_b a = 2, f(n)=n^2 = \Theta(n^2)$ → 情形 2，$T(n) = \Theta(n^2 \log n)$
- $T(n) = 2T(n/2) + n$：$a=2, b=2, \log_b a = 1, f(n)=n = \Theta(n)$ → 情形 2，$T(n) = \Theta(n \log n)$（归并排序）

### 知识点 6：最好、最坏、平均情况分析

**通俗定义**：
- **最好情况**：输入对算法"最有利"，执行操作最少
- **最坏情况**：输入对算法"最不利"，执行操作最多
- **平均情况**：在所有可能输入下的**期望**操作次数

**举例**（插入排序中的插入操作，Q0046）：

将 $x$ 插入已排好序的数组 $a[0:n]$：
```
for(i = n-1; i >= 0 && x < a[i]; i--)
    a[i+1] = a[i];
a[i+1] = x;
```

- **最好**：$x \geq a[n-1]$（比最后一个元素都大），只需比较 **1 次** → $\Theta(1)$
- **最坏**：$x < a[0]$（比第一个元素还小），需要比较 **$n$ 次** → $\Theta(n)$
- **平均**：$x$ 等可能插入 $n+1$ 个位置，平均比较次数 $\frac{1+2+\cdots+n+n}{n+1} = \frac{n}{2} + \frac{n}{n+1}$ → $\Theta(n)$

**初学易错点**：
- 平均情况不等于最好和最坏的"中间值"——需要知道输入的概率分布
- 实际工程中通常关注**最坏情况**，因为它给出了算法性能的上限保证

### 知识点 7：P、NP、NPC、NPH

**通俗定义**：
- **P 类问题**：能在**多项式时间**内（如 $O(n^2)$、$O(n^3)$）求解的问题
- **NP 类问题**：能在多项式时间内**验证**解是否正确的问题（"猜答案很快，验证答案也很快"）
- **NP 完全（NPC）问题**：NP 中最难的一类——所有 NP 问题都能多项式地约化为它
- **NP 难（NPH）问题**：不比 NPC 容易（甚至更难）的问题，不要求是 NP 的

**关系图**：
```
      ┌───────────────┐
      │      NPH      │
      │  ┌─────────┐  │
      │  │   NPC    │  │
      │  │  ┌───┐   │  │
      │  │  │NP │   │  │
      │  │  │┌─┐│   │  │
      │  │  ││P││   │  │
      │  │  │└─┘│   │  │
      │  │  └───┘   │  │
      │  └─────────┘  │
      └───────────────┘
```

**证明一个问题 A 是 NPC 的步骤**：
1. 证明 $A \in NP$（可以在多项式时间内验证解）
2. 选择一个**已知的 NPC 问题** $B$
3. 构造从 $B$ 到 $A$ 的**多项式时间约化**（即证明 $B \leq_P A$）

**多项式约化的含义**：
存在一个多项式时间的确定性算法，将问题 $P$ 的每个输入 $x$ 转换为问题 $Q$ 的输入 $T(x)$，使得 $x$ 是 $P$ 的 yes 实例 $\iff$ $T(x)$ 是 $Q$ 的 yes 实例。

**初学易错点**：
- $P \subseteq NP$（P 类问题是 NP 类的子集），但 $P = NP$ 是否成立是千禧年未解难题
- NP 难（NPH）问题**不要求**属于 NP——它"至少和 NPC 一样难"但可能更复杂
- "NP 难"比"NP 完全"的范围更大——所有 NPC 问题都是 NPH，反之不成立

## 3. 典型例题

### 例题 1（来源：[Q0067]）

**题目**：计算机算法必须具备的三个特性是（ ）
A. 可执行性、可移植性、可扩充性
B. 可执行性、确定性、有穷性
C. 确定性、有穷性、稳定性
D. 易读性、稳定性、安全性

**解答**：
算法的三个基本特性是：
1. **可执行性（Effectiveness）**：算法的每一步都必须是可行的
2. **确定性（Determinism）**：每一步的含义唯一确定，无歧义
3. **有穷性（Finiteness）**：算法必须能在有限步后结束

解析选项：
- A 选项：可移植性、可扩充性是**软件工程**的要求，不是算法的基本特性
- B 选项：三个特性都正确 ✓
- C 选项：稳定性是排序算法的特性，不是所有算法的基本特性
- D 选项：易读性、安全性是软件质量属性，不是算法定义的必要条件

**答案**：B

**解析**：这道题考查对算法定义的理解。算法的三个基本特性来自算法的经典定义（D.E.Knuth 在《计算机程序设计艺术》中的论述）。注意区分"算法"和"程序"——操作系统是程序，不是算法（因为它不满足"有穷性"）。

---

### 例题 2（来源：[Q0066]）

**题目**：设 $n$ 是描述问题规模的非负整数，下面程序片段的时间复杂度是（ ）

$$
x = 2;
$$
$$
\text{while } (x < n / 2)
$$
$$
x = 2 \cdot x;
$$

A. $O(\log n)$
B. $O(n)$
C. $O(n \log n)$
D. $O(n^2)$

**解答**：
分析循环执行次数：
- 初始 $x = 2$
- 每轮循环 $x$ 乘以 2：$x$ 的取值序列为 $2, 4, 8, 16, \ldots, 2^k$
- 循环条件为 $x < n/2$，即 $2^k < n/2$
- 两边取以 2 为底的对数：$k < \log_2 (n/2) = \log_2 n - 1$
- 所以循环执行次数约为 $\log_2 n - 1$ 次

因此时间复杂度为 $O(\log n)$。

**答案**：A

**解析**：这是典型的"倍增"模式。只要循环变量以常数倍增长（每次乘以 2、乘以 3 等），循环次数就是对数量级。类似的还有"二分查找"——每次将搜索范围减半，复杂度也是 $O(\log n)$。

### 例题 3（来源：[Q0008]）

**题目**：应用 Master 方法求解以下递归方程
（1）$T(n) = 7T(n/2) + cn^2$（5 分）
（2）$T(n) = 4T(n/2) + n^2$（5 分）

**解答**：

**（1）$T(n) = 7T(n/2) + cn^2$**

套用 Master 定理：$a = 7$，$b = 2$，$\log_b a = \log_2 7 \approx 2.81$，$f(n) = cn^2$

比较 $f(n) = cn^2$ 与 $n^{\log_b a} = n^{2.81}$：
- $f(n) = O(n^{2.81 - \varepsilon})$，取 $\varepsilon = 0.5$ 则 $n^{2.81 - 0.5} = n^{2.31}$，$cn^2 = O(n^{2.31})$ 成立
- 属于 Master 定理的**情形 1**

因此：
$$
T(n) = \Theta(n^{\log_2 7}) = \Theta(n^{2.81})
$$

**（2）$T(n) = 4T(n/2) + n^2$**

套用 Master 定理：$a = 4$，$b = 2$，$\log_b a = \log_2 4 = 2$，$f(n) = n^2$

比较 $f(n) = n^2$ 与 $n^{\log_b a} = n^2$：
- $f(n) = \Theta(n^2)$，两者同阶
- 属于 Master 定理的**情形 2**

因此：
$$
T(n) = \Theta(n^2 \log n)
$$

**解析**：Master 方法的核心是**比较 $f(n)$ 与 $n^{\log_b a}$ 的大小关系**，而不是比较 $f(n)$ 与 $n$ 或 $n^2$ 本身。初学者容易直接看 $n^2$ 就以为是情形 2，但实际上关键是比较 $f(n)$ 和 $n^{\log_b a}$。

### 例题 4（来源：[Q0010]）

**题目**：使用步计数法分析下面算法在最好和最坏情形的渐近时间复杂度（5 分）

```cpp
template <class T>
bool MinMax(T a[], int n, int & Min, int & Max)
{
    if (n < 1) return false;
    Min = Max = 0;
    for (int i = 1; i < n; i++)
        if (a[Min] > a[i]) Min = i;
        else if (a[Max] < a[i]) Max = i;
    return true;
}
```

**解答**：

用步计数法逐语句分析：

| 行号 | 语句 | s/e | 最好频率 | 最坏频率 | 最好步数 | 最坏步数 |
|------|------|-----|----------|----------|----------|----------|
| 1 | 函数声明 | 0 | 1 | 1 | 0 | 0 |
| 2 | `if (n < 1) return false;` | 1 | 1 | 1 | 1 | 1 |
| 3 | `Min = Max = 0;` | 1 | 1 | 1 | 1 | 1 |
| 4 | `for (int i = 1; i < n; i++)` | 1 | $n$ | $n$ | $n$ | $n$ |
| 5 | `if (a[Min] > a[i])` | 1 | $n-1$ | $n-1$ | $n-1$ | $n-1$ |
| 6 | `Min = i;` | 1 | 0 | $n-1$ | 0 | $n-1$ |
| 7 | `else if (a[Max] < a[i])` | 1 | $n-1$ | 0 | $n-1$ | 0 |
| 8 | `Max = i;` | 1 | 0（也可能执行） | 0 | 0 | 0 |
| 9 | `return true;` | 1 | 1 | 1 | 1 | 1 |

**说明**：
- 最好情况：数组已**降序排列**（$a[0]$ 最大，$a[1]$ 最小）。第 5 行的 `a[Min] > a[i]` 对 $i=1$ 为 true，更新 $Min$；此后每次 $a[Min] > a[i]$ 均为 false，进入 else if。假设数组也是升序排列的另一种最好情况（每个元素既大于前一个也大于 Min），也会有类似结果。
  
  更典型的"最好情况"分析：第 6 行最好时执行 0 次或常数次，第 7 行最坏执行 $n-1$ 次。

- 最坏情况：数组已**升序排列**（$a[0]$ 最小）。每次执行第 5 行时 $a[Min] > a[i]$ 均为 false（因为 $a[0]$ 始终最小），进入 `else if`，但 $a[Max] < a[i]$ 始终为 true（新元素比之前所有都大），所以每次都要更新 $Max$。第 6 行执行 $n-1$ 次。

**总步数**：
- 最好：$1 + 1 + n + (n-1) + 0 + (n-1) + 0 + 1 = 3n + 1$
  $$
  \text{最好情形时间复杂度} = \Theta(n)
  $$
- 最坏：$1 + 1 + n + (n-1) + (n-1) + 0 + 0 + 1 = 4n - 1$
  $$
  \text{最坏情形时间复杂度} = \Theta(n)
  $$

**答案**：最坏情形和最好情形的时间复杂度均为 $\Theta(n)$。

**解析**：这道题展示了步计数法的完整流程——对每条语句分别统计频率后求和。有趣的是，虽然最好和最坏的具体执行步数不同（最好约 $3n$，最坏约 $4n$），但渐近上两者都是 $\Theta(n)$——线性增长的阶数不变，只是常数因子不同。这就是"渐近分析"的意义：关注增长趋势，忽略常数差异。

### 例题 5（来源：[Q0059]）

**题目**：证明 $O(f) + O(g) = O(f + g)$（7 分）

**解答**：

**证明**：
令 $F(n) = O(f)$，根据大 $O$ 记号的定义，存在自然数 $n_1$ 和正数 $c_1$，使得对任意 $n \geq n_1$，有：
$$
F(n) \leq c_1 f(n)
$$

同理，令 $G(n) = O(g)$，存在自然数 $n_2$ 和正数 $c_2$，使得对任意 $n \geq n_2$，有：
$$
G(n) \leq c_2 g(n)
$$

取 $c_3 = \max\{c_1, c_2\}$，$n_3 = \max\{n_1, n_2\}$，则对所有的 $n \geq n_3$：
$$
F(n) \leq c_1 f(n) \leq c_3 f(n)
$$
$$
G(n) \leq c_2 g(n) \leq c_3 g(n)
$$

因此：
$$
O(f) + O(g) = F(n) + G(n) \leq c_3 f(n) + c_3 g(n) = c_3 (f(n) + g(n))
$$

根据大 $O$ 记号的定义，存在 $c_3 > 0$ 和 $n_3$，使得 $F(n) + G(n) \leq c_3 (f(n) + g(n))$，所以：
$$
O(f) + O(g) = O(f + g)
$$

证毕。

**解析**：
- 理解大 $O$ 记号的关键是"存在常数 $c$ 和阈值 $n_0$"
- 证明中取各常数的最大值（$\max$）是常见技巧，保证所有不等式同时成立
- 结论的含义：两个 $O$ 级函数的和，其 $O$ 级不会超过两者中较大的那个

### 自测题 9（来源：[Q0023]）

**提示**：这个递归式每次将规模减少 2，但有一部分固定为 $T(2)$。将递归树分层展开，注意 $T(2)$ 是常数。

### 自测题 10（来源：[Q0035]）

以下伪代码算法将 $x$ 插入到前 $n$ 个元素已排好序的数组 $a[0:n]$ 中，分析算法在最好、最坏和平均情形所用的关键字比较次数。

```
for (i ← n-1; i ≥ 0 且 x < a[i]; i--)
    a[i+1] ← a[i];
a[i+1] ← x;
```

**提示**：比较次数取决于 $x$ 在有序序列中的插入位置。最好情况是 $x$ 比所有元素都大（仅比较 1 次），最坏是 $x$ 比所有元素都小（比较 $n$ 次），平均是所有位置等可能。

> [!question]- 点击查看完整解答
> **解答**：
> 
> **最好情况**：$x \geq a[n-1]$（$x$ 大于等于最大的元素）
> - 循环条件 `x < a[i]` 第一次判断即为 false
> - 比较次数 = **1**
> 
> **最坏情况**：$x < a[0]$（$x$ 比最小的元素还小）
> - 需要从 $i=n-1$ 一直比较到 $i=0$（共 $n$ 次），然后 $i=-1$ 时条件 $i \geq 0$ 不满足而退出
> - 比较次数 = **n**
> 
> **平均情况**：
> - $x$ 有 $n+1$ 个可能的插入位置（$n$ 个元素之间有 $n+1$ 个空位）
> - 假设每个位置等可能，概率为 $1/(n+1)$
> - 插入位置 0（在最前面）：比较 $n$ 次
> - 插入位置 1：比较 $n-1$ 次
> - ...
> - 插入位置 $n-1$：比较 1 次
> - 插入位置 $n$（在最后面）：比较 1 次（与位置 $n-1$ 相同）
> 
> 平均比较次数：
> $$
> \frac{1}{n+1}(n + (n-1) + \cdots + 1 + 1) = \frac{1}{n+1}\left(\frac{n(n+1)}{2} + 1\right) = \frac{n}{2} + \frac{1}{n+1}
> $$
> 
> 渐近表示：平均时间复杂度为 $\Theta(n)$。

### 自测题 14（来源：[Q0043]）

简述渐近记号的含义。

> [!question]- 点击查看完整解答
> **解答**：
> 
> 渐近记号用于描述算法时间复杂度随输入规模 $n$ 增长时的**增长趋势**，忽略常数因子和低阶项。
> 
> - **大 $O$ 记号**：$f(n) = O(g(n))$ 表示存在正常数 $c$ 和 $n_0$，使得当 $n \geq n_0$ 时，$0 \leq f(n) \leq c g(n)$。即 $f(n)$ 的增长**不超过** $g(n)$ 的常数倍。
> - **大 $\Omega$ 记号**：$f(n) = \Omega(g(n))$ 表示存在正常数 $c$ 和 $n_0$，使得当 $n \geq n_0$ 时，$0 \leq c g(n) \leq f(n)$。即 $f(n)$ 的增长**至少**是 $g(n)$ 的常数倍。
> - **大 $\Theta$ 记号**：$f(n) = \Theta(g(n))$ 表示 $f(n) = O(g(n))$ 且 $f(n) = \Omega(g(n))$。即 $f(n)$ 与 $g(n)$ **同阶**增长。
> 
> 类比：好比比较不同汽车的速度——大 $O$ 表示"不超过"某速度，大 $\Omega$ 表示"至少"某速度，大 $\Theta$ 表示"大约"某速度。

### 自测题 16（来源：[Q0001]）

简答：
1. 简述 NP、NPH 问题和它们之间的关系。
2. 证明 $O(f(n)) + O(g(n)) = O(\min\{f(n), g(n)\})$。
3. 动态规划适用于哪些问题？简述动态规划思想。

**提示**：
- 第 1 问参考知识点 7
- 第 2 问注意结论本身；证明思路与典型例题 5 类似
- 第 3 问是后续章节的内容，可简要回答：最优子结构 + 重叠子问题

> [!question]- 点击查看完整解答
> **解答**：
> 
> **（1）NP、NPH 及其关系**
> 
> - **NP 问题**：能在多项式时间内**验证**解的正确性的问题（Non-deterministic Polynomial）。即给定一个"证书"，可以在多项式时间内判断答案是否正确。
> - **NPH（NP 难）问题**：如果所有 NP 问题都可以多项式约化到问题 $H$，则 $H$ 是 NP 难的。NPH 问题不要求属于 NP。
> 
> **关系**：
> - NPC（NP 完全问题）是 NP 和 NPH 的交集
> - NP ⊆ NPH？不一定——NP 问题不一定能多项式约化到所有 NP 问题
> - NPH 不一定是 NP 的（如停机问题是 NPH 但不是 NP 的）
> 
> **（2）$O(f(n)) + O(g(n)) = O(\min\{f(n), g(n)\})$**
>

> 
> **（3）动态规划**
> 
> - **适用问题**：具有**最优子结构**（最优解包含子问题最优解）和**重叠子问题**（子问题被重复求解）的问题
> - **基本思想**：将原问题分解为相互重叠的子问题，通过求解子问题并将结果**填表存储**，避免重复计算，从而自底向上构建原问题的最优解

$$

> [!question]- 点击查看完整解答
> **解答**：
> 
> **证明**：
> 
> 使用**代入归纳法**。
> 
> **归纳假设**：对所有小于 $n$ 的正整数 $k$，有 $T(k) \leq c_1 k^2 - c_2 k$。
> 
> **归纳步骤**（$n > 1$）：
> 
> 代入递归式：
> $$
> \begin{aligned}
> T(n) &= 4T(n/2) + n \\
> &\leq 4\left[c_1 (n/2)^2 - c_2 (n/2)\right] + n \quad \text{（由归纳假设）} \\
> &= 4\left[c_1 \frac{n^2}{4} - c_2 \frac{n}{2}\right] + n \\
> &= c_1 n^2 - 2c_2 n + n \\
> &= c_1 n^2 - c_2 n - (c_2 n - n)
> \end{aligned}
> $$
> 
> 如果取 $c_2 > 1$，则 $c_2 n - n > 0$，有：
> $$
> T(n) \leq c_1 n^2 - c_2 n - (c_2 n - n) \leq c_1 n^2 - c_2 n
> $$
> 
> **基础情形**（$n = 1$）：
> 
> 需要 $T(1) = c \leq c_1 \cdot 1^2 - c_2 \cdot 1 = c_1 - c_2$
> 
> 即 $c_1 \geq c + c_2$
> 
> **结论**：取 $c_2 > 1$ 且 $c_1 \geq c + c_2$，则对所有 $n \geq 1$，$T(n) \leq c_1 n^2 - c_2 n$ 成立。
>

### 自测题 20（来源：[Q0052]）

（1）简述算法复杂性分析的方法及用途。
（2）算法 A 的计算时间 $f(n)$ 满足递归关系式：$f(n) = 2f(n-1) + 1$；$n > 0$，$f(0) = 1$。算法 B 的计算时间 $g(n) = 2f(n/2) + \log n$。请使用渐近符号分别表示 $f(n)$ 和 $g(n)$。

**提示**：
- （1）方法有解析法和实验测量法，分析角度有时间复杂度和空间复杂度
- （2）$f(n) = 2f(n-1) + 1$ 展开是等比数列求和 → $O(2^n)$；$g(n)$ 的 $f(n/2)$ 指什么需注意
- （3）参考知识点 7

> [!question]- 点击查看完整解答
> **解答**：
> 
> **（1）算法复杂性分析的方法及用途**
> 
> **方法**：
> - **解析法**：通过数学推导计算算法执行步数的函数表达式
> - **实验测量法**：实际运行程序，统计运行时间或资源消耗
> 
> **分析角度**：
> - **时间复杂度**：运行时间随输入规模的增长趋势
> - **空间复杂度**：内存使用量随输入规模的增长趋势
> 
> **用途**：根据分析结果选择或评价算法，例如选择满足时间/空间约束的算法。
> 
> **（2）求解 $f(n)$ 和 $g(n)$**
> 
> 对于 $f(n) = 2f(n-1) + 1$，$f(0) = 1$：
> $$
> \begin{aligned}
> f(n) &= 2f(n-1) + 1 \\
> &= 2[2f(n-2) + 1] + 1 = 2^2 f(n-2) + 2 + 1 \\
> &= 2^3 f(n-3) + 2^2 + 2 + 1 \\
> &\ \vdots \\
> &= 2^n f(0) + (2^{n-1} + 2^{n-2} + \cdots + 2 + 1) \\
> &= 2^n \cdot 1 + (2^n - 1) \\
> &= 2^{n+1} - 1 = \Theta(2^n)
> \end{aligned}
> $$
> 
> 对于 $g(n) = 2f(n/2) + \log n$，其中 $f$ 是上面的函数：
> $$
> g(n) = 2 \cdot \Theta(2^{n/2}) + \log n = \Theta(2^{n/2})
> $$
> 
> 因为指数函数 $2^{n/2}$ 远大于 $\log n$，所以 $g(n) = \Theta(2^{n/2})$。
> 
> **（3）多项式约化与 NPC 证明**
> 
> **多项式约化的含义**：存在一个多项式时间的确定性算法 $T$，使得对于问题 $P$ 的每个输入 $x$，$x$ 是 $P$ 的 yes 实例 **当且仅当** $T(x)$ 是问题 $Q$ 的 yes 实例。记为 $P \leq_P Q$。
> 
> **证明问题 $Q$ 是 NPC 的步骤**：
> 1. 证明 $Q \in NP$（能在多项式时间内验证解）
> 2. 选择一个**已知的 NPC 问题** $P$
> 3. 构造从 $P$ 到 $Q$ 的**多项式时间约化**，即证明 $P \leq_P Q$

> [!question]- 点击查看完整解答
> **解答**：
> 
> 分析各层循环：
> 
> - 最外层：$m = 2, 3, \ldots, n$，共 $n-1$ 次
> - 中间层：$i = 0, 1, \ldots, n-m$，共 $n-m+1$ 次
> - 最内层：$\min_{i < l \leq j}\{ \cdots \}$ 需要检查 $j-i = m$ 个元素，时间复杂度 $O(m)$
> 
> 中间层的总执行时间（对给定的 $m$）：
> $$
> \sum_{i=0}^{n-m} O(m) = O(m \cdot (n-m+1)) = O(m(n-m))
> $$
> 
> 所有层的总执行时间：
> $$
> \begin{aligned}
> T(n) &= \sum_{m=2}^{n} O(m(n-m)) \\
> &= O\left(\sum_{m=2}^{n} (mn - m^2)\right) \\
> &= O\left(n \sum_{m=2}^{n} m - \sum_{m=2}^{n} m^2\right) \\
> &= O\left(n \cdot \frac{n(n+1)}{2} - \frac{n(n+1)(2n+1)}{6}\right) \\
> &= O\left(\frac{n^3}{2} - \frac{n^3}{3} + \text{低阶项}\right) \\
> &= O(n^3)
> \end{aligned}
> $$
> 
> 因此算法的渐进时间复杂度为 $\Theta(n^3)$。

> 这个算法对数组每对元素 $(i, j)$ 都比较一次（共 $C_n^2 = n(n-1)/2$ 对，但这里每对比较了两次，为 $(n-1)^2 \approx n^2$），因此是平方级复杂度。

### 自测题 24（来源：[Q0068]）

下列函数中渐近时间复杂度最小的是（ ）
A. $T(n) = \log_2 n + 5000n$
B. $T(n) = n^2 - 8000n$
C. $T(n) = n^3 + 5000n$
D. $T(n) = 2n \log_2 n - 1000n$

**提示**：比较各函数的主项阶数。$n < n \log n < n^2 < n^3$。

> [!question]- 点击查看完整解答
> **答案**：A
> 
> **解析**：忽略低阶项和常数系数：
> - A：主项 $5000n$，阶数 $O(n)$
> - B：主项 $n^2$，阶数 $O(n^2)$
> - C：主项 $n^3$，阶数 $O(n^3)$
> - D：主项 $2n \log_2 n$，阶数 $O(n \log n)$
> 
> 增长速率：$O(n) < O(n \log n) < O(n^2) < O(n^3)$，因此 A 最小。
