# 第 3 章 贪心法

## 1. 本章概述

贪心法（Greedy Algorithm）是一种在每一步决策中都采取当前状态下最优（最有利）的选择，从而希望最终得到全局最优解的算法设计策略。本章学习贪心法的基本思想、适用条件（贪心选择性质和最优子结构性质），并通过活动安排、连续背包、最小平均完成时间、汽车加油以及 1-优化法等经典问题，掌握贪心法的设计思想和正确性证明（尤其是交换论证法）。

---

## 2. 知识点讲解

### 2.1 贪心法的基本思想

**通俗定义**：
贪心法就像日常生活中"先做最重要或最紧急的事"的决策方式——在每个决策点上，只考虑当前情况下的最佳选择，不做长远规划，也不回头调整之前的选择。

**两个核心要素**：

1. **贪心选择性质（Greedy Choice Property）**：通过一系列局部最优选择（贪心选择），能够拼凑出全局最优解。这是贪心法成立的关键前提。
2. **最优子结构性质（Optimal Substructure）**：一个问题的最优解包含其子问题的最优解。这与动态规划的要求一致，但贪心法更强——它要求每一步选"当前最好"，而不需要像动态规划那样枚举所有子问题。

**标准流程**：

```
从问题的初始状态出发
while (能朝目标前进一步):
    选择当前状态下的最优决策
    将问题规模缩小为子问题
输出最终结果
```

**初学易错点**：
- ❌ 以为贪心法能解决所有最优化问题 → ✅ 只有在满足贪心选择性质时贪心法才保证最优
- ❌ 把贪心法和动态规划混淆 → ✅ 贪心法是"选一次就走"（每一步只做一次选择，不再回头）；动态规划是"比较所有走法后选最好的"（需要存储子问题的所有可能解）
- ❌ 以为"每一步最优"必然导致"全局最优" → ✅ 很多问题（如 0/1 背包）贪心法会得到次优解，必须用数学证明（如交换论证）来验证贪心选择的正确性

### 2.2 活动安排问题（Activity Selection Problem）

**通俗定义**：
有一批活动，每个活动有一个开始时间和结束时间。你是一个教室管理员，同一时间只能安排一个活动举办。你的目标是安排尽可能多的活动。

**贪心策略**：
每次选择**结束时间最早**且与已选活动不冲突的活动。直觉上：结束时间越早，留给后续活动的时间越多，就能安排更多的活动。

**标准算法描述**：

```
1. 将所有活动按结束时间从小到大排序
2. 选择结束时间最早的活动作为第一个
3. 依次检查后续活动：
   - 如果当前活动的开始时间 ≥ 上一个选中活动的结束时间，则选中它
   - 否则跳过
4. 重复步骤 3 直到所有活动检查完毕
```

**正确性证明思路（交换论证法）**：
假设一个最优解与贪心解在第一个选择上不同。用贪心选择的第一个活动（结束时间全局最早）替换最优解的第一个活动。由于贪心选择的活动的结束时间不晚于最优解的第一个活动，替换后仍然可行且活动数量不变。重复这一替换过程，可以将任意最优解转化为贪心解而不损失活动数量。因此贪心法得到的是活动数量最多的方案。

**初学易错点**：
- ❌ 习惯性地想按"开始时间最早"或"持续时间最短"选择 → ✅ 只有"结束时间最早"可以保证最优（可以举反例：按开始时间最早可能选到一个开始早但结束极晚的活动，挤掉后面所有活动；按持续时间最短也可能选到两个冲突的短活动而错过中间穿插的长活动）
- ❌ 忘记先排序再选择 → ✅ 贪心法通常需要先对数据排序
- ❌ 判断冲突时条件写反 → ✅ 条件为 `s[i] ≥ last_finish`（当前活动开始时间 ≥ 上一个的结束时间）

### 2.3 连续背包问题（Fractional Knapsack Problem）

**通俗定义**：
你有一个容量有限的背包，每个物品有重量和价值，但物品可以分割（比如切一块黄金）。目标是在不超过背包容量的前提下使总价值最大。

> **与 0/1 背包的关键区别**：0/1 背包中物品要么全部拿走、要么不拿（不可分割）；连续背包中物品可以只拿一部分（可分割）。这个区别使得两者的解法完全不同。

**贪心策略**：
按**价值密度**（$d_i = p_i / w_i$，即单位重量的价值）从大到小依次装入。如果当前物品能全部装下就全部装入，否则装入一部分直到背包满。

**价值密度公式**：
$$d_i = \frac{p_i}{w_i}$$

**伪代码**：

```
算法 FractionalKnapsack(w[1..n], p[1..n], c)
// 输入：物品重量 w，物品价值 p，背包容量 c
// 输出：装入比例 x[1..n]

    计算每个物品的价值密度 d[i] = p[i] / w[i]
    按 d[i] 非递增的顺序排序物品

    remain = c                     // 剩余容量
    for i = 1 to n:
        if w[i] ≤ remain:
            x[i] = 1               // 全部装入
            remain = remain - w[i]
        else:
            x[i] = remain / w[i]   // 装入一部分
            remain = 0
            break

    return x[1..n]
```

**正确性证明（核心思路）**：
设物品已按价值密度非递增排列，$x$ 是贪心解，$y$ 是最优解。找到第一个不同分量 $j$，有 $x_j > y_j$（因为贪心法尽可能多地装第 $j$ 个物品）。从 $y$ 后面物品中腾出容量补到第 $j$ 个物品上（用高价值密度物品替换低价值密度物品），不降低总价值。反复进行，最终将 $y$ 变为 $x$，故 $x$ 也是最优解。

**初学易错点**：
- ❌ 以为连续背包的贪心策略也适用于 0/1 背包 → ✅ 0/1 背包不能按价值密度贪心（有反例，见 2.4 节）
- ❌ 忘记计算价值密度而直接按价值或重量排序 → ✅ 必须综合考虑重量和价值，只看重量或只看价值都会出错
- ❌ 证明最优性时跳步 → ✅ 必须明确：替换后可行性不变、总价值不降低

### 2.4 0/1 背包问题与 1-优化法

**通俗定义**：
同样是背包问题，但这次物品不可分割——要么拿整件，要么不拿。这种约束使得简单贪心失效。

**为何简单贪心会失效**：
选择一件"价值密度高但比较重"的物品，可能挤掉了多件"价值密度略低但总价值更高"的物品组合。然而腾出来的空闲容量又不足以装下另一件合适的物品，导致总价值反而更低。

> **经典反例**：
> n = 3, w = [18, 15, 10], p = [25, 24, 15], c = 30
> - 价值密度：物品 1 ≈ 1.39, 物品 2 = 1.6, 物品 3 = 1.5
> - 按密度贪心：先选物品 2（价 24，重 15），再选物品 3（价 15，重 10），总价值 = 39
> - 但最优解为：物品 1（价 25，重 18）+ 物品 3（价 15，重 10），总价值 = 40
> - 贪心法未得到最优解！

**1-优化法（1-Optimization Method）**：
贪心法的改进策略——在简单贪心的基础上，增加"预选"步骤来扩大搜索范围。

**步骤**：
1. 按价值密度排序所有物品
2. 执行一次标准贪心（不预选任何物品），得到解 $S_0$
3. 对每个物品 $i$（$i = 1 \text{ to } n$）：强制选入物品 $i$（预选），然后对剩余物品按贪心策略取尽可能多的物品，得到解 $S_i$
4. 在所有 $S_0, S_1, ..., S_n$ 中取总价值最大的作为最终解

**时间复杂度分析**：

| 步骤 | 操作 | 时间复杂度 |
|------|------|-----------|
| 1 | 按价值密度排序 | $O(n \log n)$ |
| 2 | 不预选的贪心 | $O(n)$ |
| 3 | $n$ 次预选，每次贪心 | $n \times O(n) = O(n^2)$ |
| **总计** | | $\Theta(n^2 + n + n\log n) = \Theta(n^2)$ |

**初学易错点**：
- ❌ 误以为 1-优化法一定能得到 0/1 背包的最优解 → ✅ 1-优化法只是贪心法的改进，**仍然不保证全局最优**（它只在 k=1 的局部范围内搜索）
- ❌ 预选后忘记去掉已预选的物品再贪心 → ✅ 预选后对剩余物品执行贪心时，剩余容量应减去预选物品的重量

### 2.5 最小平均完成时间问题（Minimizing ACT）

**通俗定义**：
你有一台机器和一批任务，每个任务需要不同的执行时间。你需要决定任务的执行顺序。如果一个任务的完成时间定义为"从开始到它结束的时间"，那么平均完成时间 ACT 就是所有任务完成时间的平均值。问：什么顺序能让 ACT 最小？

**相关公式**：
假设任务顺序为 $\pi_1, \pi_2, ..., \pi_n$，任务 $\pi_k$ 的执行时间为 $t_{\pi_k}$，完成时间为：
$$c_{\pi_k} = t_{\pi_1} + t_{\pi_2} + \cdots + t_{\pi_k}$$

平均完成时间为：
$$ACT = \frac{1}{n} \sum_{k=1}^n c_{\pi_k}$$

**贪心策略（SPT 规则）**：
按任务执行时间**从小到大**的顺序执行（Shortest Processing Time First，最短处理时间优先）。

**正确性证明（交换论证法）**：
假设一个最优解中存在逆序：任务 $i$ 在前、任务 $j$ 在后，且 $t_i > t_j$。交换 $i$ 和 $j$ 的位置后，重新计算 ACT：

$$n \cdot ACT - n \cdot ACT' = (i - j)(t_j - t_i)$$

因为 $i < j$ 且 $t_i > t_j$，所以 $(i-j) < 0$、$(t_j - t_i) < 0$，上式 $> 0$，即 $ACT > ACT'$。说明交换后 ACT 变小了。因此任何含有逆序的顺序都不是最优的——最优顺序必须没有逆序，即按执行时间从小到大排列。

**初学易错点**：
- ❌ 混淆"完成时间"和"执行时间" → ✅ 第 $k$ 个任务的完成时间 = 前面所有 $k$ 个任务的执行时间之和
- ❌ 以为需要动态规划等复杂算法 → ✅ SPT 规则就是排个序，时间复杂度 $O(n \log n)$
- ❌ 交换论证中写错系数 → ✅ 第 $i$ 个任务的完成时间在 $n \cdot ACT$ 中的系数是 $(n - i + 1)$

### 2.6 汽车加油问题

**通俗定义**：
你开车从起点到终点，沿途有一些加油站。汽车加满油可以行驶 $n$ 公里。问：如何选择最少的加油站停靠加油，使你能顺利到达终点？

**输入**：
- 满油可行驶距离 $n$
- 各段距离序列 $x[1..k]$，其中 $x[i]$ 表示第 $i$ 个加油站到第 $(i+1)$ 个加油站（或终点）的距离

**输出**：最少加油次数（如果某段距离 $> n$，则无解）

**贪心策略**：
"不到万不得已不加油"——每次加油后，尽量开到当前油量能到达的最远的加油站再加油。

**伪代码**：

```
int minRefuelTimes(vector<int> x, int n):
    k = x.size()

    // 可行性检查
    for j = 0 to k-1:
        if x[j] > n:
            print "No solution"
            return -1

    // 贪心加油
    count = 0          // 加油次数
    s = 0              // 当前已行驶距离（从上一次加油后累计）

    for i = 0 to k-1:
        s = s + x[i]
        if s > n:       // 当前油量不足以到达第 i+1 站
            count++     // 在第 i 个加油站加油
            s = x[i]    // 重置（加满油后从当前段开始计算）

    return count
```

**示例**：
$n = 10$，各段距离 $x = [3, 4, 5, 2, 2, 3]$

| 步骤 | 当前段 | 累计 s | s > 10? | 动作 | 加油次数 |
|------|--------|--------|---------|------|---------|
| 1 | 3 | 3 | 否 | 继续开 | 0 |
| 2 | 4 | 7 | 否 | 继续开 | 0 |
| 3 | 5 | 12 | **是** | 在加油站 2 加油，s=5 | 1 |
| 4 | 2 | 7 | 否 | 继续开 | 1 |
| 5 | 2 | 9 | 否 | 继续开 | 1 |
| 6 | 3 | 12 | **是** | 在加油站 5 加油，s=3 | 2 |

最少加油次数：**2 次**。

**初学易错点**：
- ❌ 忘记可行性检查 → ✅ 如果任意一段距离超过 $n$，汽车无法通过，必须输出 "No solution"
- ❌ 在最后一个加油站也加油 → ✅ 如果剩余油量足够到达终点，不需要在最后一个加油站加油
- ❌ 混淆"加油站之间的距离"和"加油站的位置" → ✅ 输入是各段距离，不是绝对位置

## 3. 典型例题

### 例题 1（来源：[Q0004]）——活动安排

**题目**：
(1) 用贪心法写出活动安排的伪代码。
(2) 求 [1, 2)、[2, 4)、[6, 7)、[4, 8) 的安排方法和使用教室数量。
(3) 证明这种算法是否能得到最优解。

**解答**：

**(1) 伪代码：**

```
算法 ActivitySelection(s[1..n], f[1..n])
// 输入：活动开始时间 s[i]，结束时间 f[i]
// 输出：选中的活动集合 A

    将活动按结束时间 f[i] 从小到大排序
    // 排序后记为 s'[1..n], f'[1..n]

    A = {1}                        // 第一个活动（结束时间最早）直接选中
    last_finish = f'[1]            // 记录上一个选中活动的结束时间
    count = 1

    for i = 2 to n:
        if s'[i] ≥ last_finish:    // 不冲突
            A = A ∪ {i}
            count++
            last_finish = f'[i]    // 更新结束时间

    return A
```

**(2) 实例求解：**

四个活动及其时间段：

| 活动 | 开始 | 结束 | 是否被选中 |
|------|------|------|-----------|
| A    | 1    | 2    | ✅ 第一个选中 |
| B    | 2    | 4    | ✅ 开始时间 2 ≥ 2，不冲突 |
| C    | 6    | 7    | ❌ 开始时间 6 < 8（上一个选中 D 的结束时间），冲突 |
| D    | 4    | 8    | ✅ 开始时间 4 ≥ 4，不冲突 |

**执行过程**：
1. 按结束时间排序：A(2) < B(4) < D(8) < C(7) → 等等，按结束时间排序应该是 A(2)、B(4)、C(7)、D(8)。但题目给的活动顺序是 A[1,2)、B[2,4)、C[6,7)、D[4,8)，按结束时间排序后顺序为 A、B、C、D。
   - 正确排序：A(2) < B(4) < C(7) < D(8)
2. 选中活动 A [1, 2)，last_finish = 2
3. 检查 B [2, 4)：开始 2 ≥ 2 → 选中，last_finish = 4
4. 检查 C [6, 7)：开始 6 ≥ 4 → 选中，last_finish = 7
5. 检查 D [4, 8)：开始 4 < 7 → 冲突，跳过

所以选中活动：**A、B、C**，共 3 个活动，需要 **1 个教室**（它们互不冲突）。

> 注意：这里给出的是标准贪心的结果，即选择 A[1,2)、B[2,4)、C[6,7)。题面的排序 [1,2)、[2,4)、[6,7)、[4,8) 可能暗示按某种特定顺序输入，但只要按结束时间排序后执行贪心，结果就是 A、B、C 三个活动，共 1 个教室。

**(3) 正确性证明（交换论证法）：**

**证明目标**：按结束时间最早优先的贪心策略总能选出数量最多的活动集合。

**证明过程**：

设贪心法选出的活动序列为 $g_1, g_2, ..., g_m$，其结束时间分别为 $f_{g_1} \leq f_{g_2} \leq ... \leq f_{g_m}$。

设任意一个最优解为 $o_1, o_2, ..., o_k$（$k$ 为最优解的活动数量），其结束时间排序后为 $f_{o_1} \leq f_{o_2} \leq ... \leq f_{o_k}$，且 $k \geq m$。

**基础情况**：因为 $g_1$ 是全局结束时间最早的活动，所以 $f_{g_1} \leq f_{o_1}$。构造新序列 $g_1, o_2, o_3, ..., o_k$：
- $o_1$ 被替换为 $g_1$，但 $g_1$ 的结束时间不晚于 $o_1$，所以 $g_1$ 与 $o_2$ 之间的间隔不会小于 $o_1$ 与 $o_2$ 之间的间隔
- 新序列仍然可行（活动之间不冲突），且仍有 $k$ 个活动

**归纳步骤**：假设已经成功将前 $t$ 个活动替换为贪心选择 $(g_1, ..., g_t)$，当前序列为 $g_1, ..., g_t, o_{t+1}, ..., o_k$，且 $f_{g_t} \leq f_{o_t}$。由于 $g_{t+1}$ 是在开始时间 $\geq f_{g_t}$ 的所有活动中结束最早的，而 $o_{t+1}$ 的开始时间 $\geq f_{o_t} \geq f_{g_t}$，所以 $f_{g_{t+1}} \leq f_{o_{t+1}}$。因此可以用 $g_{t+1}$ 替换 $o_{t+1}$。

**结论**：通过上述逐步替换，可以将最优解完全转化为贪心解而不减少活动数量（$k = m$）。所以贪心法得到的是最优解。

$$■$$

**解析**：
- 活动安排问题是贪心法的经典入门案例，体现了"结束时间最早优先"的直观合理性
- 正确性证明的核心是**交换论证**——这是贪心法最常见的证明技术：假设最优解→找到差异→用贪心选择替换→证明可行性不变→证明解的质量不降→归纳完成替换
- 第 (2) 问注意题目问"使用教室数量"——由于所有选中的活动互不冲突，同一时间只有 1 个活动，故只需要 1 个教室

### 例题 2（来源：[Q0026][Q0027][Q0028]）——连续背包问题

**题目**：
考虑 $0 \leq x_i \leq 1$ 而不是 $x_i \in \{0, 1\}$ 的连续背包问题。一种可行的贪婪策略是：按价值密度非递减的顺序检查物品，若剩余容量能容下正在考察的物品，将其装入；否则，往背包中装入此物品的一部分。

(1) 对于 $n = 3$，$w = [100, 10, 10]$，$p = [20, 15, 15]$ 及 $c = 105$，上述装入方法获得的结果是什么？（5 分）
(2) 写出伪代码。（5 分）
(3) 证明这种贪心算法总能获得最优解。（10 分）

**解答**：

**(1) 实例求解：**

**第一步**：计算每个物品的价值密度

$$d_1 = \frac{20}{100} = 0.2,\quad
d_2 = \frac{15}{10} = 1.5,\quad
d_3 = \frac{15}{10} = 1.5$$

**第二步**：按价值密度从大到小排序

排序结果：物品 2 (1.5) ≥ 物品 3 (1.5) > 物品 1 (0.2)

排序后：
- $w' = [10, 10, 100]$
- $p' = [15, 15, 20]$

**第三步**：按贪心策略依次装入

| 步骤 | 当前物品 | 重量 | 价值 | 剩余容量 | 能否全装 | 动作 | 已装价值 |
|------|---------|------|------|---------|---------|------|---------|
| 1 | 物品 2 | 10 | 15 | 105 | ✅ 105 ≥ 10 | 全装，剩余 95 | 15 |
| 2 | 物品 3 | 10 | 15 | 95 | ✅ 95 ≥ 10 | 全装，剩余 85 | 30 |
| 3 | 物品 1 | 100 | 20 | 85 | ❌ 85 < 100 | 装入 85/100=0.85，剩余 0 | 30+17=47 |

**最终结果**：
$$x = (0.85, 1, 1) \quad \text{（按原始编号）}$$
总价值 = 47

**(2) 伪代码：**

```
算法 FractionalKnapsack(w[1..n], p[1..n], c)
// 输入：物品重量数组 w[1..n]
//       物品价值数组 p[1..n]
//       背包容量 c
// 输出：装入比例数组 x[1..n]（0 ≤ x[i] ≤ 1）

    // 计算价值密度
    for i = 1 to n:
        d[i] = p[i] / w[i]

    // 按价值密度非递增排序物品
    // 排序后 w'[i]、p'[i] 对应排序后的物品
    使用排序算法将物品按 d[i] 从大到小排列

    // 贪心装入
    remain = c                     // 剩余容量
    for i = 1 to n:
        if remain == 0:
            break
        if w'[i] ≤ remain:
            x[i] = 1               // 全部装入
            remain = remain - w'[i]
        else:
            x[i] = remain / w'[i]  // 装入一部分
            remain = 0

    return x[1..n]
```

**(3) 最优性证明：**

**前提假设**：设物品已按价值密度非递增排列，即 $d_1 \geq d_2 \geq ... \geq d_n$。

设 $x = (x_1, x_2, ..., x_n)$ 是贪心法得到的解（$0 \leq x_i \leq 1$），$y = (y_1, y_2, ..., y_n)$ 是任意一个最优解。目标是证明 $\sum_{i=1}^n p_i x_i = \sum_{i=1}^n p_i y_i$。

**证明**：

**情况 1**：$x = y$。此时贪心解就是最优解，结论成立。

**情况 2**：$x \neq y$。

设 $j$ 是第一个使 $x_j \neq y_j$ 的下标，即满足：
$$x_1 = y_1, x_2 = y_2, ..., x_{j-1} = y_{j-1}, \text{但 } x_j \neq y_j$$

由贪心算法的执行过程可知：算法按价值密度从大到小依次检查物品，对当前物品尽可能多地装入。因此：
- 如果算法能全装第 $j$ 个物品（$w_j \leq \text{remain}$），则 $x_j = 1$，而 $y_j \leq 1$，所以 $x_j \geq y_j$
- 如果算法只能装一部分（$w_j > \text{remain}$），则 $x_j = \text{remain}/w_j$ 是当前剩余容量能装的最大比例，而 $y_j$ 作为一个可行解在此物品上也最多装 $\text{remain}/w_j$，所以 $x_j \geq y_j$

由于 $x_j \neq y_j$，因此有 $x_j > y_j$。

因为总容量固定，两组解装入的总重量相等：
$$\sum_{i=1}^n w_i x_i = \sum_{i=1}^n w_i y_i = c$$

所以最优解在第 $j$ 个物品上少装的重量，必定被第 $j+1$ 到第 $n$ 个物品中的某些物品多装来补偿。

设 $\delta = w_j (x_j - y_j) > 0$，即第 $j$ 个物品上贪心解比最优解多装的重量。

**构造新解** $y'$：将最优解 $y$ 调整如下——
- 将 $y_j$ 增加到 $x_j$（增加了 $\delta$ 的重量）
- 从 $j$ 后面的物品中减去总计 $\delta$ 的重量（具体可以从任意一个 $y_k$（$k > j$）中扣除，或从多个物品中分摊）

由于 $j$ 后面的物品价值密度 ≤ 第 $j$ 个物品的价值密度（$d_j \geq d_{j+1} \geq ... \geq d_n$），用第 $j$ 个物品（密度更高或相等）替换后面物品（密度更低或相等）的重量，背包总价值不会降低：
$$\sum p_i y'_i \geq \sum p_i y_i$$

而 $y$ 是最优解，所以 $\sum p_i y'_i = \sum p_i y_i$，$y'$ 也是最优解。

重复这一过程，每次将最优解中第一个与贪心解不同的分量调整为一致，最终可以将任意最优解转化为贪心解而不降低总价值。因此贪心解也是最优解。

$$■$$

**解析**：
- 连续背包问题的贪心策略正确性依赖于"物品可分割"——这就允许我们在不改变总重量的前提下，把后面密度低的物品"替换"成前面密度高的物品
- 证明中使用的是**交换论证的变体**：不是一次交换两个完整物品（因为物品可能只装了一部分），而是"削高补低"——从后面挪容量给前面
- 务必注意区分"连续背包"和"0/1 背包"：本题的证明完全依赖物品可分割性，不适用于 0/1 背包

### 例题 3（来源：[Q0014]）——最小平均完成时间

**题目**：
给定 $n$ 个任务，$1, 2, ..., n$，假定任务 $i$ 要求的执行时间是 $t_i$。如果按顺序 $1, ..., n$ 执行这些任务，任务 $i$ 的完成时间 $c_i$ 为 $t_1 + t_2 + ... + t_i$。定义一个任务顺序的平均完成时间 ACT 为 $(c_1 + c_2 + ... + c_n)/n$，不同的顺序可能有不同的平均完成时间。试证明：在 $n!$ 个可能的任务顺序中，按最小执行时间优先的贪心策略得到的任务顺序有最小的平均完成时间。（10 分）

**解答**：

**证明（交换论证法）**：

按最小执行时间优先（Shortest Processing Time First，SPT）的策略得到的任务顺序，等同于将任务按执行时间从小到大排列。下面证明此顺序的 ACT 最小。

假设在某任务顺序中，存在两个任务，序号 $i < j$（即任务 $i$ 排在任务 $j$ 之前），但 $t_i > t_j$（即存在一个**逆序**——执行时间长的任务排在了执行时间短的任务前面）。

交换任务 $i$ 和任务 $j$ 的顺序，得到一个新的任务顺序。设原顺序的平均完成时间为 $ACT$，新顺序为 $ACT'$。下面证明 $ACT' < ACT$。

考察 $n \cdot ACT$ 的表达式。在原顺序中，第 $k$ 个任务的完成时间 $c_k$ 在求和 $\sum_{k=1}^n c_k$ 中的系数为 $(n - k + 1)$。这是因为 $c_k$ 出现在第 $k$ 个至第 $n$ 个任务的完成时间中，共 $(n - k + 1)$ 次。

因此：
$$n \cdot ACT = \sum_{k=1}^n c_k = nt_1 + (n-1)t_2 + \cdots + (n-i+1)t_i + \cdots + (n-j+1)t_j + \cdots$$

交换 $i$ 和 $j$ 后，新顺序中 $i$ 和 $j$ 的位置互换，其他任务不变：
$$n \cdot ACT' = nt_1 + (n-1)t_2 + \cdots + (n-i+1)t_j + \cdots + (n-j+1)t_i + \cdots$$

两式相减：
$$
\begin{aligned}
n \cdot ACT - n \cdot ACT' &= \big[(n-i+1)t_i + (n-j+1)t_j\big] - \big[(n-i+1)t_j + (n-j+1)t_i\big] \\
&= (n-i+1)(t_i - t_j) + (n-j+1)(t_j - t_i) \\
&= (n-i+1 - n + j - 1)(t_i - t_j) \\
&= (j - i)(t_i - t_j)
\end{aligned}
$$

由于 $i < j$，所以 $j - i > 0$；由于 $t_i > t_j$（逆序），所以 $t_i - t_j > 0$。因此：
$$n \cdot ACT - n \cdot ACT' > 0$$
$$ACT > ACT'$$

这说明**交换一个逆序对后，平均完成时间减小了**。

因此，任何包含逆序的任务顺序都不可能是最优的（因为总可以通过一次交换得到更小的 ACT）。最优的任务顺序必须没有任何逆序，即所有任务按执行时间从小到大排列。

所以 SPT 规则（按最小执行时间优先的贪心策略）得到的任务顺序具有最小的平均完成时间。

$$■$$

**解析**：
- 本题的证明是交换论证法的标准应用，关键是写出 $n \cdot ACT$ 的展开式并找出交换前后的差异
- 从直观理解：短任务先做能让后面的任务少等一会儿；如果让一个长任务先做，所有排在它后面的任务都要白白等待
- 实际上 $ACT$ 的最小值等价于最小化 $\sum_{i=1}^n (n-i+1)t_i$，其中系数 $(n-i+1)$ 随 $i$ 增大而减小，所以越大的 $t_i$ 应该匹配越小的系数，即大的 $t_i$ 应该排在后面——这正好是 SPT 规则
- **常见错误**：交换论证中把系数写错。牢记第 $k$ 个任务在 $n \cdot ACT$ 中的系数是 $(n-k+1)$，不是 $k$

### 例题 4（来源：[Q0063]）——汽车加油问题

**题目**：
试用贪心算法求解汽车加油问题：已知一辆汽车加满油后可行驶 $n$ 公里，而旅途中有若干个加油站。试设计一个有效算法，指出应在哪些加油站停靠加油，使加油次数最少。（12 分）

**解答**：

**问题建模**：
- 输入：满油可行驶距离 $n$，各段距离序列 $x[1..k]$（$x[i]$ 表示第 $i$ 个加油站到第 $i+1$ 个加油站或终点的距离）
- 输出：最少加油次数
- 约束：汽车只能在加油站加油，不能在路边停车

**思路分析**：
"加油次数最少"等价于"每次加油后尽量多跑路"——因为如果少跑路就加油，意味着要加更多次。所以贪心策略是：每次加满油后，一直开到当前油量能到达的最远的加油站，如果下一段距离超过剩余油量，就在当前站加油。

**算法步骤**：
1. **可行性检查**：遍历各段距离 $x[i]$，如果任意 $x[i] > n$，则汽车无法通过，输出 "No solution"
2. **贪心过程**：
   - 从起点出发，当前已行驶距离 $s = 0$，加油次数 $count = 0$
   - 遍历每一段距离 $x[i]$：
     - 将 $x[i]$ 累加到 $s$
     - 如果 $s > n$（说明油量不够开到下一站），则**在上一站**（实际上是第 $i$ 个加油站所在的位置）加油，$count + 1$，并将 $s$ 重置为 $x[i]$（加满油后从当前段开始重新计算）

**伪代码**：

```
int minRefuelTimes(vector<int> x, int n)
// 输入：各段距离 x，满油可行驶公里数 n
// 输出：最少加油次数，无解返回 -1
{
    int k = x.size();

    // 可行性检查：任何一段距离都不能超过 n
    for (int j = 0; j < k; j++)
        if (x[j] > n) {
            cout << "No solution" << endl;
            return -1;
        }

    // 贪心加油
    int count = 0;      // 加油次数
    int s = 0;          // 当前已行驶距离（从上一次加油后累计）

    for (int i = 0; i < k; i++) {
        s += x[i];
        if (s > n) {    // 当前油量不足以到达第 i+1 个加油站
            count++;    // 在第 i 个加油站加油
            s = x[i];   // 加满油后已行驶距离重置为当前段
        }
    }

    return count;
}
```

**示例验证**：
$n = 7$，各段距离 $x = [2, 3, 4, 2, 1]$

| 段 | x[i] | s(累计) | s > 7? | 动作 | count |
|----|------|---------|--------|------|-------|
| 1  | 2    | 2       | 否     | 继续 | 0 |
| 2  | 3    | 5       | 否     | 继续 | 0 |
| 3  | 4    | 9       | **是** | 在加油站 2加油, s=4 | 1 |
| 4  | 2    | 6       | 否     | 继续 | 1 |
| 5  | 1    | 7       | 否     | 到达终点 | 1 |

最少加油次数：**1 次**（在加油站 2 加油）。

**解析**：
- 汽车加油问题的贪心策略本质上是一种"最远可达"策略——每次都在保证能到达的前提下选择最远的加油站作为下一次加油点
- 正确性的直观解释：如果存在一个最优解在某个加油站 $A$ 没有加油，而贪心解在 $A$ 加油了，说明从上一个加油站出发，不加油到不了 $A$ 的下一个加油站，所以任何可行解在 $A$ 和下一个加油站之间必然有至少一次加油（可能在 $A$ 或更早的站），因此将加油点尽量后移（即贪心策略）不会增加加油次数
- 时间复杂度：$O(k)$，只需遍历一次各段距离
- 空间复杂度：$O(1)$，只需常数个变量

**(2) 时间复杂度分析：**

| 步骤 | 操作 | 时间复杂度 |
|------|------|-----------|
| 1 | 按价值密度从大到小排序 | $\Theta(n \log n)$ |
| 2 | 不预选任何物品，执行一次贪心 | $\Theta(n)$ |
| 3 | 对每个物品 $i$（共 $n$ 个）做一次预选后贪心 | $n \times \Theta(n) = \Theta(n^2)$ |
| **总计** | 排序 + 不预选贪心 + n 次预选贪心 | $\Theta(n^2 + n + n\log n) = \Theta(n^2)$ |

所以 1-优化法的时间复杂度为 **$\Theta(n^2)$**。

**解析**：
- 1-优化法是简单贪心的改进：通过每个物品"预选一次"来扩大搜索范围。如果简单贪心因为选错了第一个物品而得到次优解，1-优化法有希望用别的预选方案"绕开"这个错误
- 但 1-优化法**仍然不保证**总是能得到 0/1 背包的最优解——要保证最优需要用动态规划（$O(n \cdot c)$）或回溯法
- 在这个实例中，简单贪心本身已经找到了最优解（77），所以 1-优化法没有体现出改进效果——这正好说明对于某些实例，简单贪心也能侥幸得到最优解，但这不是普遍情况
- 注意还原到原始编号的步骤：考试时容易忘记这一步，直接输出排序后的结果

### 自测题 1（来源：[Q0040]）——连续背包问题综合

**题目**：
考虑 $0 \leq x_i \leq 1$ 而不是 $x_i \in \{0, 1\}$ 的连续背包问题。一种可行的贪婪策略是：按价值密度非递减的顺序检查物品，若剩余容量能容下正在考察的物品，将其装入；否则，往背包中装入此物品的一部分。

(1) 对于 $n = 3$，$w = [100, 10, 10]$，$p = [20, 15, 15]$ 及 $c = 105$，上述装入方法获得的结果是什么？
(2) 写出上述连续背包问题的贪心算法伪代码。
(3) 证明这种贪心算法总能获得最优解。

**提示**：本题与典型例题 2 完全一致，只是将三个小问合并为一道题。解答可参考典型例题 2。

> [!question]- 点击查看完整解答
>
> **(1) 实例求解：**
>
> 价值密度：$d_1 = 20/100 = 0.2$，$d_2 = 15/10 = 1.5$，$d_3 = 15/10 = 1.5$
>
> 按价值密度从大到小排序：物品 2、物品 3、物品 1
>
> 排序后：$w' = [10, 10, 100]$，$p' = [15, 15, 20]$
>
> 贪心过程：
> - 物品 2（重 10，价 15）：剩余 105 ≥ 10 → 全装，剩余 95
> - 物品 3（重 10，价 15）：剩余 95 ≥ 10 → 全装，剩余 85
> - 物品 1（重 100，价 20）：剩余 85 < 100 → 装入 0.85，价值 $0.85 \times 20 = 17$
>
> **结果**：$x = (0.85, 1, 1)$（按原始编号），总价值 = 15 + 15 + 17 = **47**
>
> **(2) 伪代码：**
>
> ```
> 算法 FractionalKnapsack(w[1..n], p[1..n], c)
>     for i = 1 to n:
>         d[i] = p[i] / w[i]
>     按 d[i] 非递增排序物品
>
>     remain = c
>     for i = 1 to n:
>         if remain == 0: break
>         if w[i] ≤ remain:
>             x[i] = 1
>             remain = remain - w[i]
>         else:
>             x[i] = remain / w[i]
>             remain = 0
>
>     return x[1..n]
> ```
>
> **(3) 最优性证明：**
>
> 设物品已按价值密度非递增排列，$x$ 为贪心解，$y$ 为最优解。设 $j$ 是第一个使 $x_j \neq y_j$ 的下标。
>
> 由贪心法可知 $x_j > y_j$（贪心法尽可能多地装第 $j$ 个物品）。设 $\delta = w_j(x_j - y_j) > 0$，由于总容量固定，最优解在后面的物品中多装了 $\delta$ 的重量。
>
> 构造 $y'$：将 $y_j$ 增加到 $x_j$，从后面物品中减去 $\delta$ 重量。因为 $d_j \geq d_{j+1} \geq ... \geq d_n$，用高密度替换低密度不会降低总价值，故 $y'$ 也是最优解。
>
> 重复此过程，可将最优解转化为贪心解而不降低价值，因此贪心解是最优解。
>
> $$■$$

### 自测题 3（来源：[Q0048]）——最小平均完成时间算法

**题目**：
(1) 写出上述算法的伪代码，并分析其复杂性。
(2) 证明利用该算法生成的任务序列具有最小的 ACT。

**提示**：算法即 SPT 规则（按执行时间从小到大排序）。排序 $O(n \log n)$ 加遍历 $O(n)$，总时间复杂度 $O(n \log n)$。最优性证明用交换论证法，参考典型例题 3。

> [!question]- 点击查看完整解答
>
> **(1) 伪代码：**
>
> ```
> 算法 MinACT(t[1..n])
> // 输入：各任务的执行时间 t[1..n]
> // 输出：最小平均完成时间 ACT
>
>     将任务按执行时间 t[i] 从小到大排序
>     // 排序后的顺序即为最优执行顺序
>
>     total_completion = 0
>     sum = 0
>     for i = 1 to n:
>         sum = sum + t[i]           // sum 即为第 i 个任务的完成时间
>         total_completion = total_completion + sum
>
>     ACT = total_completion / n
>     return ACT
> ```
>
> **复杂度分析**：
>
> | 步骤 | 操作 | 时间复杂度 |
> |------|------|-----------|
> | 1 | 按执行时间从小到大排序 | $O(n \log n)$ |
> | 2 | 遍历计算各任务完成时间并求和 | $O(n)$ |
> | **总计** | | $O(n \log n + n) = O(n \log n)$ |
>
> 算法的时间复杂度由排序主导，为 $O(n \log n)$。
>
> **(2) 最优性证明：**
>
> 设任务已按执行时间从小到大排序为 $t_1 \leq t_2 \leq ... \leq t_n$。在这个顺序下：
>
> $$ACT = \frac{1}{n} \sum_{i=1}^n c_i = \frac{1}{n}\big(nt_1 + (n-1)t_2 + ... + t_n\big)$$
>
> 用交换论证证明：设某个顺序中存在逆序 $i < j$ 但 $t_i > t_j$，交换 $i$ 和 $j$ 后：
>
> $$n \cdot ACT - n \cdot ACT' = (j - i)(t_i - t_j) > 0$$
>
> 即 $ACT > ACT'$，说明交换逆序能使 ACT 减小。因此任何含逆序的顺序都不最优，最优顺序必然是按 $t_i$ 从小到大排列。
>
> $$■$$

**题目**：

>

> |------|------|-----------|

> | 不预选执行一次贪心 | 遍历 $n$ 个物品 | $\Theta(n)$ |
> | $n$ 次预选，每次对剩余物品贪心 | 每次遍历 $n-1$ 个物品，共 $n$ 次 | $\Theta(n^2)$ |
> | **总计** | | $\Theta(n^2 + n + n\log n) = \Theta(n^2)$ |
>

>
> | 问题 | 贪心策略 | 能否保证最优 | 证明方法 |
> |------|---------|------------|---------|
> | 活动安排 | 结束时间最早优先 | ✅ 是 | 交换论证 |
> | 连续背包 | 价值密度最高优先 | ✅ 是 | 交换论证 |
> | 0/1 背包 | 价值密度最高优先 | ❌ 否（需 1-优化法改进） | 反例 |
> | 最小 ACT | 执行时间最短优先（SPT） | ✅ 是 | 交换论证 |
> | 汽车加油 | 最远可及优先 | ✅ 是 | 反证法 |
>
> **通用证明套路——交换论证法**：
> 1. 假设一个最优解与贪心解不同
> 2. 找到第一个分歧点
> 3. 用贪心选择替换最优解中的对应选择
> 4. 证明替换后解仍然可行且质量不降
> 5. 归纳至完全转化为贪心解
