# 第 2 章 递归法

---

## 1. 本章概述

递归法是一种将原问题逐步分解为规模更小、结构相同的子问题来求解的算法设计方法。本章学习递归的基本概念、递归方程的建立方法，以及三种求解递归方程的方法——**代入法（代入归纳法）**、**递归树法**、**Master 定理（主方法）**，并通过 Hanoi 塔等经典案例深入理解递归的执行过程与复杂度分析。

---

## 2. 知识点讲解

### 2.1 递归的基本概念

**通俗定义**：递归就是一个函数在它的定义中调用它自己。每一次调用处理一个更小（更简单）的版本，直到遇到最简单的情况（称为"递归基"或"基准情形"）为止。好比俄罗斯套娃——打开一个大娃娃，里面有一个小一点的娃娃，再打开，还有一个更小的，直到最小的那个不能再打开。

**三个要素**：

1. **递归基（Base Case）**——最小规模的子问题，能够直接求解，不再递归调用。
2. **递归步骤（Recursive Step）**——将原问题分解为更小规模的同类子问题，并调用自身去求解。
3. **递推关系（Recurrence Relation）**——描述原问题规模 $n$ 的解与子问题规模解之间的数学关系。

**一般形式**：

$$
T(n) = a\,T\!\left(\frac{n}{b}\right) + f(n)
$$

其中：
- $n$ 是问题规模，
- $a$ 是子问题的个数（每次分解产生 $a$ 个子问题），
- $b$ 是每次分解后子问题的规模缩小倍数（每个子问题的规模约为 $n/b$），
- $f(n)$ 是分解与合并的开销。

**举例**：阶乘函数

$$
\text{fact}(n) = \begin{cases}
1, & n = 0 \quad (\text{递归基}) \\[4pt]
n \times \text{fact}(n-1), & n > 0 \quad (\text{递归步骤})
\end{cases}
$$

计算 $\text{fact}(3)$ 的执行过程：
- $\text{fact}(3) = 3 \times \text{fact}(2)$
- $\text{fact}(2) = 2 \times \text{fact}(1)$
- $\text{fact}(1) = 1 \times \text{fact}(0)$
- $\text{fact}(0) = 1$（递归基，开始返回）
- 逐层回代：$\text{fact}(0)=1 \to \text{fact}(1)=1 \to \text{fact}(2)=2 \to \text{fact}(3)=6$

**初学易错点**：
- ❌ **缺少递归基**：函数会无限递归，最终栈溢出。每次写递归时，先问自己"什么时候停止？"
- ❌ **递归参数不收敛于基**：每次调用问题规模没有减小，导致死循环。要确保每次递归调用都向基条件靠拢。
- ❌ **认为递归效率一定低**：递归是一种"设计思想"，效率取决于具体实现（如尾递归可优化为迭代，分治递归如归并排序是非常高效的）。

### 2.2 递归方程的建立

将递归算法的执行时间表达为递归方程（递推式），是分析递归复杂度的第一步。

**步骤**：

1. **确定递归基的时间复杂度**：通常是常数 $O(1)$。
2. **写出递归步骤的时间开销**：分解开销 + 子问题求解时间 + 合并开销。
3. **将上述关系表示成数学方程**。

**举例**：二分查找（Binary Search）

```
binarySearch(arr, left, right, target):
    if left > right:                    // 递归基
        return -1
    mid = (left + right) // 2           // 分解（O(1)）
    if arr[mid] == target:
        return mid
    elif target < arr[mid]:
        return binarySearch(arr, left, mid-1, target)  // 1个子问题，规模 n/2
    else:
        return binarySearch(arr, mid+1, right, target) // 1个子问题，规模 n/2
```

递归方程：

$$
T(n) = T\!\left(\frac{n}{2}\right) + O(1)
$$

其中 $O(1)$ 是计算 mid 和比较的开销，$T(1) = O(1)$ 是递归基。

**初学易错点**：
- ❌ **混淆 $a$ 和 $b$**：$a$ 是子问题个数，$b$ 是规模缩小倍数。例如二分查找 $a=1$（只查一边）、$b=2$（规模减半）。
- ❌ **忽略递归基**：$T(1)=O(1)$ 是分析的边界条件，不能省略。没有递归基的递归方程是不完整的。

### 2.3 Master 定理（主方法）

Master 定理用于求解形如 $T(n) = a\,T(n/b) + f(n)$ 的递归方程，其中 $a \ge 1$，$b > 1$，$f(n)$ 是渐近正函数。

**定理内容**：

比较 $f(n)$（分解合并的开销）与 $n^{\log_b a}$（子问题的总工作量）的增长速度：

1. **情况一**：$f(n) = O\!\left(n^{\log_b a - \varepsilon}\right)$（$\varepsilon > 0$），即 $f(n)$ 的增长**慢于** $n^{\log_b a}$
   $$
   T(n) = \Theta\!\left(n^{\log_b a}\right)
   $$

2. **情况二**：$f(n) = \Theta\!\left(n^{\log_b a}\right)$，即 $f(n)$ 与 $n^{\log_b a}$ 增长**相同**
   $$
   T(n) = \Theta\!\left(n^{\log_b a} \log n\right)
   $$

3. **情况三**：$f(n) = \Omega\!\left(n^{\log_b a + \varepsilon}\right)$（$\varepsilon > 0$）且满足**正则性条件** $a\,f(n/b) \le c\,f(n)$（$c<1$），即 $f(n)$ 的增长**快于** $n^{\log_b a}$
   $$
   T(n) = \Theta\!\big(f(n)\big)
   $$

**直观理解**：$n^{\log_b a}$ 是"子问题自身的工作量"，$f(n)$ 是"分解和合并的开销"。谁增长快，谁就主导总复杂度。

**举例**：

- $T(n) = 4T(n/2) + n$：$\log_b a = \log_2 4 = 2$，$f(n)=n = O(n^{2-\varepsilon})$，属于情况一，$T(n)=\Theta(n^2)$。
- $T(n) = 4T(n/2) + n^2$：$\log_b a = 2$，$f(n)=n^2 = \Theta(n^2)$，属于情况二，$T(n)=\Theta(n^2 \log n)$。
- $T(n) = 4T(n/2) + n^3$：$\log_b a = 2$，$f(n)=n^3 = \Omega(n^{2+\varepsilon})$，且 $4 \cdot (n/2)^3 = n^3/2 \le c n^3$（$c=1/2<1$），属于情况三，$T(n)=\Theta(n^3)$。

**Master 定理的扩展（重要）**：

当 $f(n) = \Theta\!\left(n^{\log_b a} \log^k n\right)$（$k \ge 0$）时，属于情况二的扩展：

$$
T(n) = \Theta\!\left(n^{\log_b a} \log^{k+1} n\right)
$$

例如 $T(n) = 2T(n/2) + n\log n$：$a=2, b=2, \log_b a=1$，$f(n)=n\log n = n^1 \log^1 n$，$k=1$，则 $T(n) = \Theta(n \log^2 n)$。

**初学易错点**：
- ❌ **Master 定理不适用于所有递归方程**——当 $f(n)$ 与 $n^{\log_b a}$ 的比值不是 $n^\varepsilon$ 的多项式级别时（比如比值是 $\log n$），不属于三种基本情况，而要用扩展形式。
- ❌ **忘记检查正则性条件**：情况三要求 $a\,f(n/b) \le c\,f(n)$（$c<1$），不满足时 Master 定理失效。
- ❌ **对 $T(n) = T(n-1) + ...$ 的形式使用 Master 定理**：Master 定理要求规模每次除以常数 $b$（$b>1$），$n-1$ 不是 $n/b$ 的形式，必须用迭代法。
- ❌ **将 $\log_b a$ 中的 $b$ 与代码中的常数混淆**：$\log_b a$ 是数学表达式，$b$ 是规模缩小倍数。

### 2.4 递归树法

**通俗定义**：将递归方程画成一棵树——每个节点代表一层递归调用的开销，各层开销之和就是总时间复杂度。

**步骤**：

1. **画出递归树**：根节点为 $f(n)$，每个子节点为下一层递归的 $f(n/b)$。
2. **计算每一层的总开销**：该层所有节点的开销之和。
3. **计算树的高度（递归深度）**：从根到叶子需要多少层。
4. **将所有层的开销相加**，得到总复杂度。

**举例**：$T(n) = 2T(n/2) + n$

| 层数 | 节点数 | 每个节点开销 | 该层总开销 |
|------|--------|-------------|-----------|
| 第 0 层（根） | 1 | $n$ | $n$ |
| 第 1 层 | 2 | $n/2$ | $n$ |
| 第 2 层 | 4 | $n/4$ | $n$ |
| ... | ... | ... | ... |
| 第 $k$ 层 | $2^k$ | $n/2^k$ | $n$ |

树的高度：$\log_2 n$ 层（从 $n$ 缩减到 1 需要除以 2 的次数）。

总开销：
$$
T(n) = n \times (\log_2 n + 1) = \Theta(n \log n)
$$

**初学易错点**：
- ❌ **忽略叶子节点（递归基）的开销**：到了叶子层（规模为 1），每个叶子是 $O(1)$，叶子数量可能很多（如 $T(n)=2T(n/2)+O(1)$ 有 $n$ 个叶子，总开销 $O(n)$），不能忽略。
- ❌ **求和时混淆层数和节点数**：每层有多个节点，该层总开销 = 节点数 × 每个节点开销，不要只算一个节点。
- ❌ **树的高度计算错误**：从 $n$ 到 1，每次除以 $b$，高度是 $\log_b n$；每次减 1，高度是 $n$。

### 2.5 代入法（代入归纳法）

**通俗定义**：先猜出 $T(n)$ 的大致形式（如 $O(n \log n)$），然后用数学归纳法证明这个猜测是对的。

**步骤**：

1. **猜测** $T(n)$ 的渐近形式（如 $O(n \log n)$）。
2. **归纳假设**：假设对所有小于 $n$ 的规模猜测成立。
3. **代入** 递归方程，证明对规模 $n$ 也成立。
4. **确定常数** 使不等式成立。

**举例**：$T(n) = 2T(\lfloor n/2 \rfloor) + n$，证明 $T(n) = O(n \log n)$。

**第一步（猜测）**：存在 $c>0$ 使 $T(n) \le c n \log n$。

**第二步（归纳假设）**：对 $\lfloor n/2 \rfloor$，有 $T(\lfloor n/2 \rfloor) \le c \lfloor n/2 \rfloor \log(\lfloor n/2 \rfloor)$。

**第三步（代入证明）**：

$$
\begin{aligned}
T(n) &\le 2 \left(c \frac{n}{2} \log \frac{n}{2}\right) + n \\
     &= c n (\log n - \log 2) + n \\
     &= c n \log n - c n + n \\
     &= c n \log n - (c - 1)n
\end{aligned}
$$

取 $c \ge 1$，则 $-(c-1)n \le 0$，所以 $T(n) \le c n \log n$，归纳成立。

**第四步（检验递归基）**：还需要验证 $n=1$（或较小的 $n$）时猜测成立。通常取足够大的 $c$ 即可满足基条件。

**初学易错点**：
- ❌ **猜测方向错误**：证明上界用"$\le$"，证明下界用"$\ge$"，不要把不等号弄反。
- ❌ **忘记处理取整**：实际递归中 $n/2$ 有取整，代入法通常忽略取整，但要确认这一点不影响渐近结果（可以证明当 $n$ 足够大时取整影响可忽略）。
- ❌ **归纳步骤中常数选择不当**：需要保留足够的自由度来消去额外项（如上面例子中的 $-(c-1)n$ 项）。
- ❌ **忘记验证递归基**：即使归纳步骤成立，也需要确保基条件成立。

### 2.6 Hanoi 塔问题

**问题描述**：有三根柱子 A、B、C。A 上有 $n$ 个大小不等的圆盘，按大小顺序叠放（大盘在下，小盘在上）。目标：将所有圆盘从 A 移到 C，每次只能移动一个盘，且任何时候大盘不能放在小盘上面。

**递归解法**：

```
Hanoi(A, C, n):
    if n == 1:
        move(A → C)
    else:
        Hanoi(A, B, n-1)   // 将 n-1 个盘从 A 移到 B（借助 C）
        move(A → C)          // 将最大的盘从 A 移到 C
        Hanoi(B, C, n-1)   // 将 n-1 个盘从 B 移到 C（借助 A）
```

**递归方程**：

$$
T(n) = \begin{cases}
1, & n = 1 \\[4pt]
2\,T(n-1) + 1, & n > 1
\end{cases}
$$

**求解（迭代展开法）**：

$$
\begin{aligned}
T(n) &= 2T(n-1) + 1 \\
     &= 2(2T(n-2) + 1) + 1 = 4T(n-2) + 2 + 1 \\
     &= 2^2 T(n-2) + (2^2 - 1) \\
     &\quad \vdots \\
     &= 2^k T(n-k) + (2^k - 1)
\end{aligned}
$$

当 $n-k = 1$ 即 $k = n-1$ 时，代入 $T(1) = 1$：

$$
\begin{aligned}
T(n) &= 2^{n-1} \cdot T(1) + (2^{n-1} - 1) \\
     &= 2^{n-1} + 2^{n-1} - 1 \\
     &= 2^n - 1
\end{aligned}
$$

所以 $T(n) = \Theta(2^n)$。

**为什么是 $2^n - 1$？**
- 每次移动 $n$ 个盘子需要：先把 $n-1$ 个盘子移到辅助柱（$T(n-1)$ 步），再移动最大盘（1 步），最后把 $n-1$ 个盘子移到目标柱（$T(n-1)$ 步）。
- 展开后得到 $T(n) = 2^{n-1} + 2^{n-1} - 1 = 2^n - 1$。
- 例如：$n=1$ 需要 1 步，$n=2$ 需要 3 步，$n=3$ 需要 7 步，$n=64$ 需要 $2^{64}-1 \approx 1.84 \times 10^{19}$ 步（假设每秒移动一个盘，需要约 5849 亿年）。

**初学易错点**：
- ❌ **混淆柱子角色**：每次递归调用中，源柱、目标柱、辅助柱的角色都会变化，不要死记柱子的物理编号，而要理解"从哪里搬到哪里、借助哪根"。
- ❌ **误用 Master 定理**：Hanoi 问题的 $n$ 是逐次减 1（$n-1$），而不是除以常数 $b$，因此不能使用 Master 定理，必须用迭代法。
- ❌ **认为递归一定高效**：Hanoi 问题的指数级复杂度说明"递归"只是一种设计方法，不一定高效——当每次递归只减少常数规模时，容易导致指数爆炸。

### 2.7 递归的时间复杂度总结

常见递归方程及其解（建议熟记）：

| 递归方程 | 求解方法 | 复杂度 | 典型算法 |
|---------|---------|--------|---------|
| $T(n) = T(n-1) + O(1)$ | 迭代展开 | $\Theta(n)$ | 线性递归（如遍历链表） |
| $T(n) = T(n-1) + O(n)$ | 迭代展开 | $\Theta(n^2)$ | 选择排序、冒泡排序 |
| $T(n) = 2T(n-1) + O(1)$ | 迭代展开 | $\Theta(2^n)$ | Hanoi 塔 |
| $T(n) = 2T(n-1) + O(n)$ | 迭代展开 | $O(n \cdot 2^n)$ | 带线性开销的指数递归 |
| $T(n) = T(n/2) + O(1)$ | Master 情况二 | $\Theta(\log n)$ | 二分查找 |
| $T(n) = 2T(n/2) + O(1)$ | Master 情况一 | $\Theta(n)$ | 二叉树遍历 |
| $T(n) = 2T(n/2) + O(n)$ | Master 情况二 | $\Theta(n \log n)$ | 归并排序、快速排序（平均） |
| $T(n) = T(n/2) + O(n)$ | Master 情况三 | $\Theta(n)$ | 未优化的分治 |
| $T(n) = 4T(n/2) + O(n)$ | Master 情况一 | $\Theta(n^2)$ | 某些分治算法 |
| $T(n) = 4T(n/2) + O(n^2)$ | Master 情况二 | $\Theta(n^2 \log n)$ | Strassen 矩阵乘法（部分） |

**关键区分**：
- **规模减常数**（如 $n-1$）：递归深度为 $n$，不能使用 Master 定理，需迭代展开。
- **规模减比例**（如 $n/2$）：递归深度为 $\log n$，可以尝试 Master 定理。

## 3. 典型例题

---

**例题 1（来源：[Q0044]）**

**题目**：
应用 Master 方法解递归方程：
$$
T(n) = 5\,T\!\left(\frac{n}{5}\right) + n \log n
$$

**解答**：

**第一步**：确定标准形式的参数。

将递归方程写成 $T(n) = a\,T(n/b) + f(n)$ 的形式：
- $a = 5$（每次分解产生 5 个子问题）
- $b = 5$（每个子问题规模缩小到原来的 $1/5$）
- $f(n) = n \log n$（分解和合并的开销）

**第二步**：计算 $\log_b a$。

$$
\log_b a = \log_5 5 = 1
$$

因此 $n^{\log_b a} = n^{1} = n$。

**第三步**：比较 $f(n)$ 与 $n^{\log_b a}$ 的增长速度。

$$
f(n) = n \log n = n \cdot \log n
$$

$f(n)$ 与 $n^{\log_b a} = n$ 的比值是 $\log n$。这不是 $n^\varepsilon$ 的形式（$\log n$ 增长慢于任何 $n^\varepsilon$），因此不属于三种基本情况，而是属于**情况二的扩展**。

情况二的扩展形式：当 $f(n) = \Theta(n^{\log_b a} \log^k n)$（$k \ge 0$）时，有：

$$
T(n) = \Theta(n^{\log_b a} \log^{k+1} n)
$$

这里 $f(n) = n \log n = n^{1} \log^{1} n$，对应 $k = 1$。

**第四步**：代入扩展公式得到结果。

$$
T(n) = \Theta(n \log^2 n)
$$

**解析**：
- Master 定理的三种基本情况覆盖了 $f(n)$ 与 $n^{\log_b a}$ 的**多项式级别**比较（差一个 $n^\varepsilon$ 因子）。
- 当 $f(n) = n^{\log_b a} \log^k n$ 时，$f(n)$ 与 $n^{\log_b a}$ 的比值是 $\log^k n$，它介于情况一和情况二之间（因为 $\log^k n$ 比任何 $n^\varepsilon$ 增长得慢，但又比常数增长得快），需要用扩展形式。
- 另一种验证方法——递归树：根节点开销 $n \log n$，下一层 5 个子节点每个开销 $(n/5) \log(n/5)$，该层总开销 $n \log(n/5)$。随层数增加，总开销逐层递减（因为 $\log(n/5) < \log n$），但层数为 $\log_5 n$。对各层求和后可得到 $\Theta(n \log^2 n)$。

**例题 2（来源：[Q0047]）**

**题目**：
分析求解 Hanoi 问题的算法的渐近复杂度：

```txt
void Hanoi(A, C, n) {
    if n == 1
        move(A, C);
    else {
        Hanoi(A, B, n - 1);
        move(A, C);
        Hanoi(B, C, n - 1);
    }
}
```

**解答**：

**第一步**：写出递归方程。

- 当 $n = 1$ 时，只需一次移动，$T(1) = 1$。
- 当 $n > 1$ 时：
  - 先将 $n-1$ 个盘子从 A 移到 B（借助 C）：需要 $T(n-1)$ 步。
  - 将最大的盘子从 A 直接移到 C：需要 1 步。
  - 再将 $n-1$ 个盘子从 B 移到 C（借助 A）：需要 $T(n-1)$ 步。

所以递归方程为：

$$
T(n) = \begin{cases}
1, & n = 1 \\[4pt]
2\,T(n-1) + 1, & n > 1
\end{cases}
$$

**第二步**：迭代展开求解。

$$
\begin{aligned}
T(n) &= 2\,T(n-1) + 1 \\
     &= 2\big(2\,T(n-2) + 1\big) + 1 \\
     &= 4\,T(n-2) + 2 + 1 \\
     &= 2^2\,T(n-2) + (2^2 - 1) \\
     &\quad \vdots \\
     &= 2^k\,T(n-k) + (2^k - 1)
\end{aligned}
$$

当 $n-k = 1$ 即 $k = n-1$ 时代入 $T(1) = 1$：

$$
\begin{aligned}
T(n) &= 2^{n-1} \cdot T(1) + (2^{n-1} - 1) \\
     &= 2^{n-1} + 2^{n-1} - 1 \\
     &= 2^n - 1
\end{aligned}
$$

**第三步**：得到渐近复杂度。

$$
T(n) = 2^n - 1 = \Theta(2^n)
$$

Hanoi 问题的时间复杂度是指数级的 $\Theta(2^n)$。

**解析**：
- Hanoi 问题的递归方程 $T(n) = 2T(n-1) + 1$ 的规模参数是逐次减 1（$n-1$），而不是除以常数 $b$，因此**不能使用 Master 定理**。必须用迭代法（展开法）求解。
- 从代码结构上看：$n$ 个盘子的 Hanoi 问题需要 $2^n - 1$ 步。$n=1$ 时 1 步，$n=2$ 时 3 步，$n=3$ 时 7 步，随着 $n$ 的增大步数呈指数增长。
- 这个例子说明递归算法不一定高效——当每次递归只减少常数规模时（$n-1$），多层嵌套会导致指数爆炸。Hanoi 问题在 $n$ 较大时（如 $n=64$）在理论上是不可解的（$2^{64} \approx 1.84 \times 10^{19}$ 步）。

**例题 3（补充：归并排序的递归分析）**

**题目**：
归并排序算法将长度为 $n$ 的数组分成两半，递归排序每一半，然后在线性时间内合并。请建立递归方程并用 Master 定理解出时间复杂度。

**解答**：

**第一步**：建立递归方程。

- 分割：将数组分为两半，开销 $O(1)$。
- 递归：对左右两半分别排序，每半规模 $n/2$，两个子问题，开销 $2\,T(n/2)$。
- 合并：将两个有序子数组合并为一个，需要 $\Theta(n)$ 的时间（依次比较两数组头部元素，取较小的放入结果）。

综合以上：

$$
T(n) = 2\,T\!\left(\frac{n}{2}\right) + \Theta(n)
$$

递归基：$T(1) = \Theta(1)$（只有一个元素，已经有序）。

**第二步**：套用 Master 定理。

- $a = 2$，$b = 2$
- $\log_b a = \log_2 2 = 1$，所以 $n^{\log_b a} = n$
- $f(n) = \Theta(n)$，与 $n^{\log_b a} = n$ 同阶

命中**情况二**（$f(n) = \Theta(n^{\log_b a})$）：

$$
T(n) = \Theta\left(n^{\log_b a} \log n\right) = \Theta(n \log n)
$$

**解析**：
- 归并排序是分治递归的经典代表。Master 定理的情况二告诉我们：当分解+合并的开销与子问题总个数所决定的基本工作量相当时，总复杂度是 $n^{\log_b a} \log n$。
- 也可以用递归树验证：每一层的总开销均为 $\Theta(n)$，共有 $\log_2 n$ 层，总开销 $\Theta(n \log n)$。
- 对比：如果 $f(n) = \Theta(n^2)$（合并开销是平方级），将落入情况三，$T(n) = \Theta(n^2)$。这说明归并排序的高效依赖于 $f(n)$ 是线性的——一旦合并开销过大，整体复杂度就会退化。

---

**例题 4（补充：二分查找的递归分析）**

**题目**：
二分查找在有序数组中查找目标值，每次与中间元素比较后只在半边递归搜索。请建立递归方程并分析时间复杂度。

**解答**：

**第一步**：建立递归方程。

- 比较中间元素与目标值：$O(1)$（一次比较操作）。
- 递归搜索左半边或右半边（只选一边）：1 个子问题，规模 $n/2$。

$$
T(n) = T\!\left(\frac{n}{2}\right) + O(1)
$$

递归基：$T(1) = O(1)$（直接比较是否相等即可得出结果）。

**第二步**：套用 Master 定理。

- $a = 1$，$b = 2$
- $\log_b a = \log_2 1 = 0$，所以 $n^{\log_b a} = n^0 = 1$
- $f(n) = O(1) = \Theta(1) = \Theta(n^0)$

$f(n) = \Theta(n^{\log_b a})$，命中**情况二**：

$$
T(n) = \Theta(n^{\log_b a} \log n) = \Theta(1 \cdot \log n) = \Theta(\log n)
$$

**解析**：
- 二分查找的特殊之处在于 $a=1$（每次只处理一个子问题，不是分治而是"减治"）。$n^{\log_b a}=n^0=1$，意味着子问题自身的工作量不随 $n$ 增长。
- 总复杂度由递归深度 $\log_2 n$ 决定，每一层 $O(1)$ 开销，所以是 $O(\log n)$。
- 这是"减治"（Decrease and Conquer）而非"分治"（Divide and Conquer）的典型——虽然也是递归，但只求解一个子问题而非多个。减治往往能得到对数级或线性级复杂度。

**例题 5（补充：斐波那契数列的朴素递归）**

**题目**：
朴素递归求解斐波那契数列 $F(n) = F(n-1) + F(n-2)$，递归基 $F(1)=F(2)=1$。请写出递归方程并分析时间复杂度。

**解答**：

**第一步**：建立递归方程。

```txt
fib(n):
    if n == 1 or n == 2:
        return 1
    else:
        return fib(n-1) + fib(n-2)
```

每个 $n>2$ 的调用产生两个子问题，规模分别为 $n-1$ 和 $n-2$，合并开销 $O(1)$。

递归方程：
$$
T(n) = T(n-1) + T(n-2) + O(1)
$$

递归基：$T(1) = T(2) = O(1)$。

**第二步**：求解上界。

这个方程不是标准的 $a T(n/b) + f(n)$ 形式，不能使用 Master 定理。我们可以先用一个较松的上界来估计。

因为 $T(n-2) \le T(n-1)$（规模更大的子问题不会比规模更小的子问题耗时更少），所以：

$$
T(n) = T(n-1) + T(n-2) + O(1) \le 2\,T(n-1) + O(1)
$$

对不等式 $T(n) \le 2\,T(n-1) + c$ 展开：

$$
\begin{aligned}
T(n) &\le 2\,T(n-1) + c \\
     &\le 2\,(2\,T(n-2) + c) + c = 4\,T(n-2) + 2c + c \\
     &\le 2^k\,T(n-k) + c(2^{k} - 1)
\end{aligned}
$$

当 $n-k=2$ 时 $k=n-2$，$T(2)=1$：

$$
T(n) \le 2^{n-2} + c(2^{n-2} - 1) = O(2^n)
$$

所以 $T(n) = O(2^n)$。

**更精确的分析**：

实际上 $T(n)$ 与斐波那契数 $F(n)$ 同阶，而 $F(n) \approx \frac{1}{\sqrt{5}} \phi^n$（$\phi \approx 1.618$ 是黄金分割比），因此精确复杂度为 $\Theta(\phi^n)$，约 $\Theta(1.618^n)$。

**解析**：
- 朴素斐波那契递归之所以慢，是因为大量**重复计算**——$T(n)$ 的递归树包含 $2^n$ 量级的节点，但很多节点是完全重复的（如 $\text{fib}(3)$ 被 $\text{fib}(5)$ 和 $\text{fib}(4)$ 分别计算）。
- 这也是动态规划要解决的问题——通过**记忆化（Memoization）**，将已算过的结果存下来，遇到相同子问题时直接取用，可将复杂度降为 $O(n)$。
- 与 Hanoi 问题对比：Hanoi 的 $T(n)=2T(n-1)+1$ 是严格的 $2^n$，而斐波那契是两个不同规模的子问题（$n-1$ 和 $n-2$），其递归树形态不完全平衡，实际略小于 $2^n$（约 $1.618^n$）。

### 自测题 4

**题目**：
用**递归树法**求解 $T(n) = T(n/3) + T(2n/3) + O(n)$ 的时间复杂度。

**提示**：
- 递归树不是完全平衡的。两条路径分别为 $n \to n/3 \to n/9 \to \cdots$ 和 $n \to 2n/3 \to 4n/9 \to \cdots$。
- 树的高度由**最长路径**决定（每次都走 $2n/3$ 分支的那条）。
- 每层总开销约 $O(n)$（因为 $n/3 + 2n/3 = n$）。
- 思考：最长路径每次缩减为原来的 $2/3$，从 $n$ 到 $1$ 需要多少层？

> [!question]- 点击查看完整解答
>
> **第一步**：分析树的结构。
>
> - 根节点：开销 $n$，分出两个子节点 $n/3$ 和 $2n/3$。
> - 第 1 层：两个节点，开销分别为 $n/3$ 和 $2n/3$，总开销 = $n/3 + 2n/3 = n$。
> - 第 2 层：四个节点（$n/9$、$2n/9$、$2n/9$、$4n/9$），总开销 = $n/9 + 2n/9 + 2n/9 + 4n/9 = n$。
> - 继续展开，每一层的总开销基本保持为 $n$（因为每对兄弟节点的开销之和始终为父节点开销）。
>
> **第二步**：确定树的高度。
>
> 树的高度由最长的路径决定——即每次都走 $2n/3$ 分支的那条路径。
>
> 该路径上问题规模的变化：$n \to \frac{2}{3}n \to \left(\frac{2}{3}\right)^2 n \to \cdots \to \left(\frac{2}{3}\right)^h n = 1$。
>
> 所以：
>
> $$
> \left(\frac{2}{3}\right)^h n = 1 \quad \Rightarrow \quad h = \log_{3/2} n
> $$
>
> **第三步**：求和。
>
> 总复杂度 = 各层开销之和：
>
> $$
> T(n) = \sum_{i=0}^{h} O(n) = O\big(n \cdot (\log_{3/2} n + 1)\big) = O(n \log n)
> $$
>
> 因此 $T(n) = O(n \log n)$。
>
> （注：可以更精确地证明 $T(n) = \Theta(n \log n)$，因为不仅上界是 $n \log n$，下界也是 $n \log n$ 量级。）

### 自测题 5

**题目**：
使用**代入法**证明 $T(n) = 2\,T(n/2) + n$ 的解为 $T(n) = O(n \log n)$。

**提示**：
- 猜测 $T(n) \le c n \log n$（$c$ 为某正常数）。
- 假设 $T(n/2) \le c \cdot (n/2) \cdot \log(n/2)$。
- 代入方程展开，看看需要 $c$ 满足什么条件才能消去多余项。

> [!question]- 点击查看完整解答
>
> **第一步**：猜测。
>
> 猜测存在常数 $c > 0$，使得 $T(n) \le c n \log n$。
>
> **第二步**：归纳假设。
>
> 对 $m < n$（特别是 $m = n/2$），假设 $T(m) \le c m \log m$。
>
> 因此：
>
> $$
> T(n/2) \le c \cdot \frac{n}{2} \cdot \log \frac{n}{2}
> $$
>
> **第三步**：代入递归方程。
>
> $$
> \begin{aligned}
> T(n) &= 2\,T(n/2) + n \\
>      &\le 2 \cdot \left(c \cdot \frac{n}{2} \cdot \log \frac{n}{2}\right) + n \\
>      &= c n \cdot (\log n - \log 2) + n \\
>      &= c n \log n - c n + n \\
>      &= c n \log n - (c - 1)n
> \end{aligned}
> $$
>
> **第四步**：确定常数 $c$。
>
> 要使 $T(n) \le c n \log n$ 成立，需要：
>
> $$
> c n \log n - (c - 1)n \le c n \log n
> $$
>
> 即：
>
> $$
> -(c - 1)n \le 0 \quad \Rightarrow \quad c \ge 1
> $$
>
> 取 $c = 1$，则 $T(n) \le n \log n$ 对 $n \ge 2$ 成立。
>
> **第五步**：验证递归基。
>
> 对于 $n=1$，$T(1) = O(1)$，而 $1 \cdot \log 1 = 0$，所以 $T(1) \le 1 \cdot \log 1$ 不成立。但渐近记号的归纳证明通常对 $n \ge n_0$ 成立即可（取 $n_0=2$，然后单独处理 $n=1$ 的情况——取更大的 $c$ 可以覆盖基条件）。
>
> **第六步**：结论。
>
> 由数学归纳法，$T(n) = O(n \log n)$ 得证。

### 自测题 7

**题目**：
写出以下递归算法的时间复杂度递归方程，并用一种方法求解。

```txt
function fun(n):
    if n <= 1:
        return 1
    for i = 1 to n:
        print(i)
    return fun(n-1) + fun(n-1)
```

**提示**：
- 递归基 $T(1)=O(1)$。
- for 循环从 1 到 $n$，是 $O(n)$ 的操作。
- 递归调用产生了两个规模为 $n-1$ 的子问题。
- 得到的递归方程是 $T(n) = 2\,T(n-1) + O(n)$。比 Hanoi 问题多了一个 $O(n)$。
- 这种形式不能用 Master 定理（因为 $n-1$ 不是 $n/b$），只能用迭代展开。

> [!question]- 点击查看完整解答
>
> **第一步**：建立递归方程。
>
> - 递归基：$T(1) = O(1)$
> - for 循环：打印 1 到 $n$，$\Theta(n)$
> - 两个递归调用：各需 $T(n-1)$ 时间
>
> $$
> T(n) = 2\,T(n-1) + \Theta(n)
> $$
>
> **第二步**：迭代展开求解。
>
> $$
> \begin{aligned}
> T(n) &= 2\,T(n-1) + n \\
>      &= 2\big(2\,T(n-2) + (n-1)\big) + n \\
>      &= 4\,T(n-2) + 2(n-1) + n \\
>      &= 8\,T(n-3) + 4(n-2) + 2(n-1) + n \\
>      &\quad \vdots \\
>      &= 2^{k}\,T(n-k) + \sum_{i=0}^{k-1} 2^{i} \cdot (n-i)
> \end{aligned}
> $$
>
> 当 $n-k=1$ 即 $k=n-1$ 时：
>
> $$
> \begin{aligned}
> T(n) &= 2^{n-1} \cdot T(1) + \sum_{i=0}^{n-2} 2^{i} \cdot (n-i) \\
>      &= O(2^{n-1}) + \sum_{i=0}^{n-2} 2^{i} \cdot (n-i)
> \end{aligned}
> $$
>
> 求和部分中，$i=0$ 的项是 $n \cdot 2^0 = n$，$i=n-2$ 的项是 $2 \cdot 2^{n-2} = 2^{n-1}$。最大的项是 $i=0$ 附近，因为 $2^i$ 增长快于 $(n-i)$ 的衰减，总和由最大的项 $n \cdot 2^{n-1}$ 主导。
>
> 所以：
>
> $$
> T(n) = O(n \cdot 2^n)
> $$
>
> **结论**：$T(n) = O(n \cdot 2^n)$，比 Hanoi 问题（$2^n$）多了一个 $n$ 因子，因为每次递归调用前还有 $\Theta(n)$ 的循环开销。

| **Master 情况一** | $f(n)$ 增长慢于 $n^{\log_b a}$ → $T(n) = \Theta(n^{\log_b a})$ |
| **Master 情况二** | $f(n)$ 与 $n^{\log_b a}$ 同阶 → $T(n) = \Theta(n^{\log_b a} \log n)$ |
| **Master 情况二扩展** | $f(n) = \Theta(n^{\log_b a} \log^k n)$ → $T(n) = \Theta(n^{\log_b a} \log^{k+1} n)$ |
| **Master 情况三** | $f(n)$ 增长快于 $n^{\log_b a}$（且满足正则性条件） → $T(n) = \Theta(f(n))$ |
| **递归树法** | 每层开销求和，适合 $a$、$b$ 不规则的方程 |
| **代入法** | 猜 + 数学归纳法验证，适合任意形式的递归方程 |
| **Hanoi 塔** | $T(n)=2T(n-1)+1$，$\Theta(2^n)$，不能用 Master，用迭代法 |
| **指数递归 vs 分治递归** | 规模减常数（$n-1$）→ 指数级；规模减比例（$n/b$）→ 对数/线性/线性对数级 |

**常见误区速查**：

1. ❌ 对 $T(n)=T(n-1)+...$ 的形式套用 Master 定理 → 应使用迭代法（$n-1$ 不是 $n/b$）。
2. ❌ 将 $\log_b a$ 中的 $b$ 与代码中的常数混淆 → $\log_b a$ 是数学表达式，底数为 $b$。
3. ❌ 忘记检查情况三的正则性条件 → 不满足时 Master 定理失效。
4. ❌ 忽略递归树的叶子节点总开销 → 叶子数量可能很多（如 $2^{\log_2 n} = n$ 个），不能忽略。
5. ❌ 认为"递归"一定低效 → 分治递归（如归并排序 $\Theta(n \log n)$）往往是最优解；指数级递归（如 Hanoi、朴素斐波那契）才是低效的。
6. ❌ 搞混 $a$ 和 $b$ 的含义 → $a$ 是子问题个数，$b$ 是规模缩小倍数。例如二分查找：$a=1$（只搜一边），$b=2$（规模减半）。
