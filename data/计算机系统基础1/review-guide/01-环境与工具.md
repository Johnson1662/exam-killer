本章介绍计算机系统实验和开发中必备的**环境与工具**，包括 Linux 基本命令、GCC 编译器、Vim 编辑器、SSH 远程连接、GDB 调试器和 Git 版本控制工具。掌握这些工具是后续学习编译、链接、调试等内容的基础。

## 2. 知识点讲解

### 2.1 Linux 基本命令

#### 2.1.1 文件与目录操作命令

| 命令 | 英文全称 | 功能 | 示例 |
|:---:|:---:|:---|:---|
| `ls` | list | 列出当前目录下的文件和子目录名 | `ls -l` 以详细格式列出 |
| `mkdir` | make directory | 创建新目录 | `mkdir mylab` 创建名为 mylab 的目录 |
| `cd` | change directory | 改变当前工作目录 | `cd /home/user` 切换到 /home/user |
| `cp` | copy | 复制文件或目录 | `cp main.c mylab/` 将 main.c 复制到 mylab |
| `rm` | remove | 删除文件或目录 | `rm file.txt` 删除文件；`rm -r dir/` 删除目录 |
| `mv` | move | 移动文件/目录，或重命名 | `mv old.txt new.txt` 重命名 |

> **通俗定义**：Linux 没有图形化"双击"操作，所有对文件和目录的管理都要通过命令完成。`ls` 相当于"打开文件夹看看里面有什么"，`mkdir` 相当于"新建文件夹"，`cd` 相当于"双击进入某个文件夹"，`cp` 相当于"复制粘贴"，`rm` 相当于"删除"。

**举例**：
当前目录为 `/home/user`，目录下有一个 `main.c` 文件，需要将其复制到 `/home/user/mylab` 目录下（该目录尚不存在）。应先后执行：
```bash
mkdir mylab       # 先创建 mylab 目录
cp main.c mylab/  # 再将 main.c 复制进去
```

> **初学易错点**：
> - `cp` 复制目录时必须加 `-r`（recursive，递归）参数，否则报错：`cp: -r not specified; omitting directory`。
> - `rm` 删除目录也必须加 `-r`，且删除不可撤销，操作前务必确认。
> - `mkdir` 与 `mk` 不同，Linux 中没有 `mk` 命令。

#### 2.1.2 find 命令与通配符匹配

`find` 命令用于在目录树中搜索文件。基本语法：

```bash
find [搜索路径] [匹配条件]
```

**通配符**：
- `?` — 匹配**任意一个**字符
- `*` — 匹配**任意多个**字符（含0个）

**举例**：
> 搜索当前目录及子目录下所有文件名为 5 个字母（不包括后缀）、且第三个字母是 a、后缀不限的文件：

```bash
find ./ -name ??a??.*
```

解释：
- `find ./` — 从当前目录开始搜索
- `-name` — 按文件名匹配
- `??a??` — 前两个任意字符，第三个必须是 a，后两个任意字符（共 5 个字符的文件名主体）
- `.` — 表示任意一个后缀字符（但 `?` 只匹配一个字符，而后缀可能多于一个字符，如 `.txt` 是 4 个字符）
- `.*` — 用 `*` 匹配任意扩展名（包括可能没有扩展名的情况）

> **初学易错点**：
> - `?` 匹配**恰好一个**字符，`*` 匹配**任意多个**字符（包括 0 个）。例如 `??a??.???` 要求后缀恰好 3 个字符，而 `??a??.*` 对后缀长度不做限制。
> - 通配符在 find 的 `-name` 参数中需要用引号或反斜杠转义，防止被 shell 先展开。上述命令中的 `??a??.*` 被引号包裹（虽然没有写引号，但在实际使用中应加引号：`"??a??.*"`）。
> - `find .` 和 `find ./` 效果相同，`.` 表示当前目录，`./` 也是当前目录。

#### 2.1.3 文件权限

Linux 中每个文件有 **9 个权限位**，分为三组，每组 3 位：

| 所属用户（owner） | 所属组（group） | 其他用户（others） |
|:---:|:---:|:---:|
| rwx | rwx | rwx |

- **r**（read，4）— 读取权限
- **w**（write，2）— 写入权限
- **x**（execute，1）— 执行权限

用 `ls -l` 查看，例如 `-rwxr-xr--` 表示：
- 所有者可读可写可执行
- 所属组可读可执行
- 其他人只可读

> **初学易错点**：不是 3 位也不是 6 位也不是 10 位，是 **9 位**。`ls -l` 输出的第一列第一个字符表示文件类型（如 `-` 为普通文件，`d` 为目录），不计算在权限位中。

### 2.2 GCC 编译器

GCC（GNU Compiler Collection）是 Linux 下最常用的 C/C++ 编译器。将源代码变为可执行文件通常经历四个阶段：

```
源代码（.c）→ 预处理 → 编译 → 汇编 → 链接 → 可执行文件
```

| 阶段 | GCC 选项 | 输出文件 | 说明 |
|:---:|:---:|:---|:---|
| 预处理 | `gcc -E` | `.i` 文件 | 展开宏、处理 `#include` |
| 编译 | `gcc -S` | `.s` 文件（汇编代码） | 将 C 代码转为汇编指令 |
| 汇编 | `gcc -c` | `.o` 文件（目标文件/机器码） | 将汇编代码转为二进制机器码 |
| 链接 | `gcc`（无特殊选项） | 可执行文件（默认 a.out） | 将目标文件与库链接为可执行文件 |

**举例**：
- `gcc -S sum.c` — 将 `sum.c` 编译为汇编代码（输出 `sum.s`）
- `gcc -c sum.c` — 将 `sum.c` 编译为目标文件（输出 `sum.o`）
- `gcc -o program sum.c` — 直接生成可执行文件，命名为 `program`

> **初学易错点**：
> - `-S`（大写 S）生成汇编**代码**（文本文件，.s），`-c`（小写 c）生成机器**码**（二进制文件，.o）。两者是完全不同的阶段产物。
> - `-o` 用于指定输出文件名，不是"生成目标文件"的选项。
> - `gcc -o sum.c` 是错误的写法，`-o` 后应跟输出文件名，而不是源代码文件名。

### 2.3 Vim 编辑器

Vim 是 Linux 终端下的文本编辑器，没有鼠标菜单，全靠键盘操作。初学者最需要掌握 3 种模式之间的切换：

```
┌──────────┐     i / a / o     ┌──────────┐
│ 普通模式 │ ──────────────→ │ 插入模式 │
│ (默认)   │ ←────────────── │ (编辑)   │
└──────────┘    Esc 键       └──────────┘
     │                              │
     │  :                             Esc
     ▼                              ▼
┌──────────┐
│ 命令模式 │
│ (末行)   │
└──────────┘
```

- **普通模式（Normal Mode）** — 启动 Vim 后的默认模式，可以跳转到其他所有模式。在此模式下按 `i` 进入插入模式，按 `:` 进入命令模式。
- **插入模式（Insert Mode）** — 在此模式下可以输入和修改文本内容。按 `Esc` 返回普通模式。
- **命令模式（Command Mode，也称末行模式）** — 在普通模式下按 `:` 进入，可输入各种命令（如保存、退出、搜索替换）。

**常用命令**：

| 操作 | 按键序列 |
|:---|:---|
| 保存并退出 | `Esc` → `:wq` → `Enter` |
| 放弃修改并退出 | `Esc` → `:q!` → `Enter` |
| 仅保存不退出 | `Esc` → `:w` → `Enter` |
| 仅退出不保存（无修改时） | `Esc` → `:q` → `Enter` |

> **初学易错点**：
> - 退出 Vim 时必须**先按 `Esc` 回到普通模式**，再输入命令。在插入模式下直接输入 `:wq` 只会把 `:wq` 当作文本输入到文件中。
> - `:wq`（write and quit）保存并退出；`:q!`（quit force）强制放弃修改退出。两者含义不同，不能混淆。
> - `:!q` 是错误的写法，`:!` 在 Vim 中用于执行外部 shell 命令，不会退出 Vim。

### 2.4 SSH 远程连接

SSH（Secure Shell）是一种加密的网络协议，用于安全地远程登录到另一台计算机。

进行 SSH 连接时需要配置：

| 需要配置 | 说明 |
|:---|:---|
| **IP 地址** | 目标主机的网络地址 |
| **端口号** | SSH 服务监听的端口（默认 22） |
| **账号** | 登录目标主机的用户名 |

**不需要配置**：编译器。编译器是远程主机上已经安装好的软件，连接 SSH 后直接可以使用，不属于连接配置项。

> **初学易错点**：SSH 只是提供了一条安全通道让你登录到远程机器，登录后能做什么取决于远程机器上装了什么软件。编译器、调试器等是远程机器上的工具，不是 SSH 连接本身需要配置的。

**常用命令示例**：
```bash
ssh username@192.168.1.100 -p 22
```

### 2.5 GDB 调试器

GDB（GNU Debugger）是 Linux 下最常用的程序调试工具。常用命令：

| 命令 | 含义 |
|:---|:---|
| `b`（break） | 设置断点 |
| `r`（run） | 运行程序 |
| `c`（continue） | 继续运行到下一个断点 |
| `n`（next） | 单步执行（跳过函数） |
| `s`（step） | 单步执行（进入函数） |
| `q`（quit） | 退出 GDB |
| `p`（print） | 打印变量值 |

> **初学易错点**：断点命令是 `b`（break 的缩写），而不是 `c`（continue）、`r`（run）或 `q`（quit）。记忆方法：break = "打破"程序执行流，所以用 `b`。

### 2.6 Git 版本控制

**通俗定义**：Git 就像代码的"时光机"。每次修改代码后，可以用 Git **提交**（commit）一个版本，之后可以随时回到任意历史版本，也可以多人协作时合并各自的修改。

**Git 的主要作用**：**管理代码**（版本控制）。它不测试、不编辑、不编译代码。

**常用命令**：

| 命令 | 功能 |
|:---|:---|
| `git clone <仓库地址>` | 将远程仓库中的代码复制到本地（首次获取代码） |
| `git push` | 将本地提交推送到远程仓库 |
| `git log` | 查看提交历史 |
| `git restore` | 恢复文件到某个历史版本 |
| `git commit -m "描述"` | 提交当前修改到本地仓库 |
| `git add <文件>` | 将修改加入暂存区 |

> **初学易错点**：
> - `git clone` 是**从远程复制到本地**，`git push` 是**从本地推送到远程**。方向不能搞反。
> - Git 只管理你告诉它要管理的文件（通过 `git add` 添加），不是目录下所有文件自动被管理。

## 3. 典型例题

### 例题 1（来源：[Q0004]）

**题目**：
现有 C 语言源代码文件 `sum.c`，下面哪一条命令可以将 `sum.c` 编译为汇编代码？

A. `gcc -c sum.c`
B. `gcc -o sum.c`
C. `gcc -S sum.c`
D. `gcc -E sum.c`

**解答**：

正确答案是 **C**：`gcc -S sum.c`

推理过程：
1. 题目要求将 C 源代码转换为**汇编代码**（即 `.s` 文件），这对应编译流程中的"编译"阶段。
2. GCC 的各个选项含义如下：
   - **`-S`**（大写 S）→ 编译到汇编阶段停止，输出 `.s` 汇编代码文件 → **符合题意**
   - `-c`（小写 c）→ 继续到汇编阶段之后，输出 `.o` 目标文件（二进制机器码），不是汇编代码
   - `-o` → 指定输出文件名，后面必须跟文件名，`gcc -o sum.c` 语法错误（因为没告诉 GCC 输入文件是什么）
   - `-E` → 只进行预处理，输出 `.i` 文件，还没有到编译阶段

**解析**：
这道题考察 GCC 编译的四个阶段及其对应选项。关键是区分 `-S`（生成汇编代码）和 `-c`（生成目标文件）。可以这样记忆：**S = assembly**（S 联想汇编语言中的段），**c = compile to object**（c 联想 object 中的 c）。此外要注意 `-o` 只是用来命名输出文件的，它本身不决定在哪个阶段停止。

### 例题 2（来源：[Q0029]）

**题目**：
`/home/user` 目录下存在一个文件 `main.c`，需要将 `main.c` 文件复制到 `/home/user/mylab` 目录下（该目录当前不存在），应该执行哪些命令才能完成上述工作？（当前工作目录为 `/home/user`）

**解答**：

需要依次执行以下两条命令：

```bash
mkdir mylab        # 第一步：创建 mylab 目录
cp main.c mylab/   # 第二步：将 main.c 复制到 mylab 中
```

推理过程：
1. 当前工作目录已经是 `/home/user`，所以 `main.c` 就在当前目录下，不需要 `cd` 切换。
2. 目标目录 `mylab` **不存在**，而 `cp` 命令无法自动创建目标目录——它只能把文件复制到**已存在**的目录中。
3. 所以必须先使用 `mkdir mylab` 创建 `mylab` 目录。
4. 然后使用 `cp main.c mylab/` 将文件复制进去（`mylab/` 末尾的斜杠不是必需的，但加上更清晰，表明目标是一个目录）。

**解析**：
这道题考察 Linux 下最基本的文件操作命令，是实际开发中非常常见的场景。易错点是初学者可能以为 `cp` 会自动创建不存在的目录，或以为 `mkdir` 不需要提前执行。实际 Linux 中，复制文件到不存在的目录会报错 `No such file or directory`，必须先 `mkdir`。

### 例题 3（来源：[Q0035]）

**题目**：
Linux 中表示搜索当前目录及子目录下所有文件名为 5 个字母（不包括后缀），且第三个是 a，后缀不限的文件，命令是：

A. `find . -name \*\*a\*\*.?`
B. `find . -name \*\*a\*\*.\*`
C. `find ./ -name ??a??.?`
D. `find ./ -name ??a??.\*`

**解答**：

正确答案是 **D**：`find ./ -name ??a??.\*`

推理过程：
1. **搜索路径**：`find ./` 和 `find .` 都表示从当前目录开始搜索，两者均可。本题重点是匹配模式。
2. **文件名主体**：要求文件名 5 个字母，第三个是 a。
   - `?` 匹配**恰好一个**字符
   - `??a??` → 第1、2个字符任意，第3个必须是 a，第4、5个任意 → **恰好 5 个字符，第三个是 a** ✓
   - `\*\*a\*\*` 中的 `*` 匹配**任意多个**字符（包括0个），所以 `\*\*a\*\*` 能匹配 "a"、"ab"、"abcde" 等任意长度 → 不符合"5个字母"要求 ✗
3. **后缀部分**：要求"后缀不限"。
   - `.*` → `.` 匹配一个点字符，`*` 匹配点后面的任意多个字符 → 能匹配 `.txt`、`.c`、`.h` 等任意后缀 ✓
   - `.?` → `.` 匹配点，`?` 匹配恰好**一个**字符 → 只能匹配后缀长度为 1 的文件（如 `.c` 可以，`.txt` 不行）✗
4. 选项 C 的 `??a??.?` 要求后缀恰好 1 个字符，不符合"后缀不限"。

因此 D 正确。

**解析**：
这道题考察 `find` 命令中的通配符使用。核心区别在于 `?` 和 `*` 的匹配规则：`?` = 正好一个字符，`*` = 零个或多个字符。易错点是以为 `*` 只能匹配多个字符（其实也能匹配 0 个），以及混淆了 `*a*` 能匹配任意包含 a 的字符串与 `??a??` 能匹配正好 5 个字符且第三个是 a 的不同含义。另外注意实际使用时最好加引号：`find ./ -name "??a??.*"`。

### 例题 4（来源：[Q0034]）

**题目**：
Linux 中下列功能的命令分别是：列出文件名和子目录名、创建目录、改变工作目录、复制文件、删除文件。

A. `ls`、`mkdir`、`cd`、`cp`、`rm`
B. `ls`、`mk`、`cd`、`cp`、`rm`
C. `dir`、`mk`、`cd`、`mv`、`rmdir`
D. `ls`、`mkdir`、`mv`、`cp`、`rmdir`

**解答**：

正确答案是 **A**：`ls`、`mkdir`、`cd`、`cp`、`rm`

逐一验证：
- **列出文件名和子目录名** → `ls`（list）✓
- **创建目录** → `mkdir`（make directory）✓
- **改变工作目录** → `cd`（change directory）✓
- **复制文件** → `cp`（copy）✓
- **删除文件** → `rm`（remove）✓

为什么不选其他选项：
- B 中的 `mk` 不是标准 Linux 命令，正确的是 `mkdir`。
- C 中的 `dir` 在 Linux 中也可用（是 `ls` 的别名），但 `mk` 不存在，`mv` 是移动不是复制，`rmdir` 只能删除**空目录**不能删文件。
- D 中的 `mv` 是移动/重命名不是复制，`rmdir` 只能删空目录不能删文件。

**解析**：
这道题考察 Linux 基本命令的名称和功能对应关系。最容易混淆的是 `cp`（复制）vs `mv`（移动），以及 `rm`（删除任意文件/目录，加 `-r`）vs `rmdir`（只能删除空目录）。记忆方法："cp = copy"，"mv = move"，"rm = remove"。

### 例题 5（来源：[Q0032]）

**题目**：
在终端中启动 Vim 后，如果在插入模式下已经修改了文件内容，如何才能在保存已修改的内容基础上安全退出 Vim？

A. 按 `Esc` 进入普通模式 → 输入 `:!q` → 回车
B. 按 `Esc` 进入普通模式 → 输入 `:q!` → 回车
C. 不需要进入普通模式直接输入 `:wq` → 回车
D. 按 `Esc` 进入普通模式 → 输入 `:wq` → 回车

**解答**：

正确答案是 **D**。

操作步骤：
1. **按 `Esc` 键**：从插入模式退出到普通模式。如果不先按 `Esc`，你输入的 `:wq` 会被当作文件内容的一部分写入，而不是被 Vim 解释为命令。
2. **输入 `:wq`**：在普通模式下按 `:` 进入命令模式，然后输入 `wq`。`w` 代表 write（写入保存），`q` 代表 quit（退出）。
3. **按回车**：执行该命令。

选项分析：
- A ❌：`:!q` 在 Vim 中表示执行外部命令 `q`（shell 命令），不会保存或退出 Vim。
- B ❌：`:q!` 表示**放弃修改**强制退出（force quit），不会保存已修改的内容。
- C ❌：在插入模式下直接输入 `:wq` 会被当作文本输入到文件中，Vim 不会将其解释为命令。
- D ✅：正确操作，先退到普通模式，再输入 `:wq` 保存并退出。

**解析**：
这道题考察 Vim 的模式切换。Vim 的核心设计理念是"模式编辑"——同一个按键在不同模式下功能不同。退出 Vim 时的标准操作顺序是：**回到普通模式 → 发出命令**。`wq` = write + quit，`q!` = force quit without saving，两者含义截然不同。`:!` 是执行外部命令的前缀，不是退出命令。

## 4. 自测题

### 自测题 1（来源：[Q0031]）

**题目**：设置 SSH 连接的时候，不用设置的是（）
A. IP 地址
B. 端口号
C. 编译器
D. 账号

> **提示**：SSH 只是建立安全通道，连接后能使用什么工具取决于远程主机。想一想：编译器是你连接时"带过去"的，还是远程主机上本来就有的？

> [!question]- 点击查看完整解答
> **答案**：C（编译器）
>
> **解析**：SSH 连接需要知道连接到哪里（IP 地址）、走哪个门进去（端口号）、你是谁（账号）。编译器是远程主机上已安装的软件工具，登录 SSH 后直接在远程主机上使用即可，不需要在 SSH 连接配置中设置。就好比你用门禁卡进入一间办公室——门禁需要验证你的身份（账号）、知道哪个门（端口号）、门在哪里（IP），但办公室里有什么办公用品（编译器）不需要在门禁系统里配置。

### 自测题 2（来源：[Q0033]）

**题目**：在终端中启动 Vim 后，如果在插入模式下已经修改了文件内容，如何放弃已做的修改并退出 Vim？
A. 按 `Esc` 进入普通模式 → 输入 `:wq` → 回车
B. 不需要进入普通模式直接输入 `:wq` → 回车
C. 按 `Esc` 进入普通模式 → 输入 `:q!` → 回车
D. 按 `Esc` 进入普通模式 → 输入 `:!q` → 回车

> **提示**：对比上一道典型例题（Q0032），那道题是"保存并退出"，这道题是"放弃修改退出"。想一想 `w`（write，写入）和 `!`（force，强制）在 Vim 命令中各代表什么含义。

> [!question]- 点击查看完整解答
> **答案**：C
>
> **解析**：
> - A ❌：`:wq` 是保存并退出，与题目要求"放弃已做的修改"矛盾。
> - B ❌：在插入模式下直接输入命令不会被 Vim 识别，需要先按 `Esc` 回到普通模式。
> - C ✅：先按 `Esc` 回到普通模式，再输入 `:q!` 强制退出。`q` = quit（退出），`!` = force（强制），表示"我不保存，直接退出"。
> - D ❌：`:!q` 是执行外部 shell 命令 `q`（shell 中通常没有这个命令），不会退出 Vim。
>
> **记忆技巧**：
> - `:wq` → write and quit → 保存并退出
> - `:q!` → quit forcefully → 强制退出（不保存）
> - 冒号前缀 `!` 是"强制执行"，跟在 `q` 后面就是"强制退出不保存"；跟在 `w` 后面是 `:w!`（强制覆盖写入）。

### 自测题 4（来源：[Q0037]）

**题目**：Linux 中每个文件有几个权限位？（）
A. 3
B. 6
C. 10
D. 9

> **提示**：Linux 权限分为三组：所有者（owner）、所属组（group）、其他用户（others），每组有 3 个权限（rwx）。总权限位数 = 组数 × 每组位数。

> [!question]- 点击查看完整解答
> **答案**：D（9 位）
>
> **解析**：Linux 文件权限共 9 位，分为三组：
> | 所有者（owner） | 所属组（group） | 其他用户（others） |
> |:---:|:---:|:---:|
> | r w x | r w x | r w x |
>
> 每组 3 位，共 3 × 3 = 9 位。注意 `ls -l` 输出的第一个字符（如 `-` 表示文件，`d` 表示目录）是文件类型标识，不属于权限位。所以 10 个字符中第 2-10 位是权限位，共 9 个。

### 自测题 5（来源：[Q0038]）

**题目**：VIM 编辑器中有 5 个模式，其中能够跳转到其他模式的是（）
A. 普通
B. 查找
C. 插入
D. 命令

> **提示**：Vim 启动后默认进入什么模式？从这个模式出发，按 `i` 能到插入模式，按 `:` 能到命令模式，按 `/` 能到查找模式。哪个模式的"枢纽"作用最强？

> [!question]- 点击查看完整解答
> **答案**：A（普通模式）
>
> **解析**：Vim 的 5 种模式分别是：普通模式（Normal）、插入模式（Insert）、命令模式（Command）、可视模式（Visual）、查找模式（Search）。其中**普通模式是进入 Vim 后的默认模式，也是唯一可以跳转到其他所有模式的模式**：
> - 普通模式 → 按 `i` → 插入模式
> - 普通模式 → 按 `:` → 命令模式
> - 普通模式 → 按 `v` → 可视模式
> - 普通模式 → 按 `/` → 查找模式
>
> 而从其他模式回到普通模式都要按 `Esc`。所以可以理解为：**普通模式是 Vim 的"中转枢纽"**。

**题目**：Git 中将远程库中的代码复制到本地的命令是（）
A. `git log`
B. `git push`
C. `git clone`
D. `git restore`

>

### 自测题 7（来源：[Q0040]）

**题目**：Git 的主要作用是（）
A. 测试代码
B. 管理代码
C. 编辑代码
D. 编译代码

> **提示**：Git 是"版本控制系统"，想象一个"代码时光机"——它能记录每次修改、让你回到任意历史版本、多人协作。它会不会代替编译器或 IDE？

> [!question]- 点击查看完整解答
> **答案**：B（管理代码）
>
> **解析**：

> - Git **管理代码版本** ✅（版本控制，记录修改历史，协作开发）
> - Git 不编辑代码（那是编辑器/IDE 的工作）
> - Git 不编译代码（那是编译器的工作）
>
> Git 的核心功能是**版本控制**，即管理代码的不同版本，方便回溯历史、多人协作、分支开发等。不要跟编译器或集成开发环境的功能搞混。
